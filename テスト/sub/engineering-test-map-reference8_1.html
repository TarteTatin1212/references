01

測れるもの・測れないもの

MEASURABLE OR NOT

「測る」という行為には常に「測れる範囲」と「測れない範囲」があります。テスト品質の世界も同じです。数値化できる側面（行数・分岐数・実行時間）と数値化できない側面（仕様の妥当性・将来の柔軟性）が同居しています。まずこの区別から始めましょう。

measurable 測れるもの・測れないものの区別
// 2つの領域
測れるもの測れないもの
カバレッジ率テストの妥当性
テスト件数テストの意義
実行時間仕様の正しさ
失敗率未来の変更容易性
ミューテーション殺害率「書かれなかったテスト」の不在

右側は本質的に重要だが数字にできない。だから左側で代用する。これがすべての出発点。

// 「測る」の根本問題 我々が本当に知りたいこと: ・このコードはバグがないか? ・想定外の事態に強いか? ・将来も信頼できるか? ↓ しかしこれらは 直接測れない ↓ 仕方なく代理指標を使う: ・カバレッジ ・テスト件数 ・失敗の検出履歴 ↓ これは「代用品」である 本質ではない // この前提を忘れると // 数字に振り回される
用語の整理
代理指標（proxy metric）＝本当に測りたいものを直接測れない時、その近似として使う指標。カバレッジは「品質」の代理指標。
定量化（quantification）＝数値として測ること。
定性的（qualitative）＝数値ではなく性質として捉えること。テストの「良さ」の多くは定性的領域に属する。
02

量と質の対立

QUANTITY vs QUALITY

テストの世界に常につきまとうのが「量」と「質」の対立です。多ければ良いわけではなく、少なくて良いわけでもない。テストにおける量と質の関係を構造的に整理しておくと、本編Vol.8で出てきたカバレッジ問題が見通しよくなります。

quantity vs quality 量と質を分けて考える
// 2軸で見る理想と危険
量 ↓ 質 ↓ テスト少・浅い
何も守れていない
量 ↑ 質 ↓ テスト多・浅い
数字上は安心、実態は不安
量 ↓ 質 ↑ テスト少・深い
局所的にしか守れない
量 ↑ 質 ↑ テスト多・深い
理想形

「量だけ」「質だけ」は危険。両方を求めるのが現実解。ただし両方を完璧にするには労力が要る。

// 量と質の役割 量 (カバレッジ・件数): 検証範囲の広さを担保 抜けがないかを示す 測りやすい 質 (検証の深さ): 検証の中身の濃さを担保 バグを実際に捕まえる能力 測りにくい ↓ 両者を バランスさせる のが テスト戦略の核心 // 量が支配的になる症状 カバレッジ目当てのテスト 検証なしの実行 90%だが意味のないテスト // 質が支配的になる症状 一部にだけ濃いテスト 他は何も検証されていない バランスの欠如
POINT
本編Vol.8で「ミューテーションテスト」が紹介された理由がここで見えます。カバレッジは「量」、ミューテーション率は「質」を測る。両方を組み合わせて、ようやくテストの実力が見えてくるのです。
03

カバレッジが示す/示さない

WHAT COVERAGE TELLS

カバレッジは何を示し、何を示さないのか。「実行されたか」は示せる。「正しく検証されたか」は示せない。この限界を構造的に理解することが、カバレッジと健全に付き合う第一歩です。

what coverage shows カバレッジの守備範囲
// 示すもの・示さないもの
示せる示せない
テストが実行した行テストが検証した内容
未通過の死角通過したが緩いテスト
分岐の網羅分岐の中身の正しさ
絶対不足絶対十分

「不足を示せる」が「十分を示せない」。これがカバレッジ指標の本質的な非対称性。

// カバレッジが嘘をつく例 function isAdult(age) { return age >= 18; } // テスト1 test("成人判定", () => { isAdult(25); // ↑ 実行はする // ↑ しかし結果を検証していない }); // カバレッジ: 100% // バグ検出力: 0% ↓ カバレッジは「通った」しか言わない 「正しいか」は判断していない // もしisAdultをreturn falseに // 変えてもこのテストは通る // → 完全に無価値なテスト // → だがカバレッジは100% // ミューテーションテストなら // この罠を暴ける (Vol.8参照)
この非対称性の意味
カバレッジは「テストが足りない場所」を高い精度で教えてくれるが、「テストが十分か」は教えてくれない。これは原理的な限界であって、技術の進歩で解決する問題ではない。不足の発見には使え、充足の保証には使えない、と覚えておけば誤用しません。
04

グッドハートの法則

GOODHART'S LAW

経済学・統計学に「グッドハートの法則」と呼ばれる経験則があります。「指標が目標になると、もはや良い指標ではなくなる」。カバレッジを目標化すると、まさにこの現象が起きます。本編Vol.8の「カバレッジ目当てのテスト」が量産される構造的理由です。

Goodhart's law 指標を目標にすると壊れる
// 法則の本文
GOODHART'S LAW When a measure becomes a target, it ceases to be a good measure.
指標が目標になると、それは良い指標でなくなる

1975年、英国の経済学者Charles Goodhartが指摘。あらゆる分野で観察される。教育のテスト点・経営のKPI・科学のh-index、そしてソフトウェアのカバレッジ。

// テスト世界での具体例 最初: カバレッジ = 品質の代理指標 品質が高いとカバレッジも高い → 良い指標 ↓ 「カバレッジ80%を目標に」と命令 ↓ 人は数字を上げに行動する: ・検証なしのテストを書く ・assertを弱める ・通る簡単なケースだけ追加 ↓ 結果: カバレッジ = 80% 達成 しかし品質は上がっていない ↓ 指標と本質が乖離する = グッドハートの法則の実例
この法則を知っておくと
「カバレッジ80%強制」が品質改善につながらない理由が、組織論ではなく構造論として理解できます。「目標にせず、観察対象として使う」が処方箋。「指標を持つことは良い。しかし目標にした瞬間に壊れる」—この鉄則は他のあらゆる指標にも当てはまります。
05

限界費用と最適点

MARGINAL COST

なぜカバレッジ100%を目指してはいけないのか。「限界費用」という経済学の概念で説明できます。同じ1%を上げるのに、序盤と終盤で必要なコストがまったく違うのです。投資対効果の曲線を理解しておきましょう。

marginal cost 「次の1%」のコストは均一ではない
// コスト/効果の曲線

80%付近に最適点。100%に近づくほどコストが急増。

// 限界費用の計算 0% → 50%: 必要工数 = 標準 ハッピーパスを書くだけ 50% → 80%: 必要工数 = やや増 異常系を追加 80% → 90%: 必要工数 = 急増 エッジケースを探す必要 90% → 95%: 必要工数 = さらに急増 到達困難な分岐を扱う 95% → 100%: 必要工数 = 事実上 無限 自動生成コード・到達不能パス ↓ 最適点 = 80% 付近 // その先は別のテストを書いた方が // 全体の品質に貢献する
用語の整理
限界費用（marginal cost）＝「もう1単位を追加するのに必要なコスト」。テストの世界では「カバレッジを次の1%上げるのに必要な工数」。
収穫逓減（diminishing returns）＝同じ投資でも、追加していくほど効果が薄くなる現象。テスト品質も典型的にこの性質を持つ。
最適点（optimal point）＝投資対効果が最大になる地点。これより手前は不足、これより先は浪費。
06

ミューテーションテストの意味

MUTATION TESTING REVISITED

本編Vol.8で紹介されたミューテーションテストは、「量」を見るカバレッジに対して「質」を見る指標です。これがなぜ生まれたのか、何を解決するのか、構造的に整理してみましょう。

mutation meaning 質を測る発想の登場
// カバレッジとミューテーションの対比
観点カバレッジ
測るもの実行された範囲
強み速い・簡単
弱み検証の質を見ない
観点ミューテーション
測るもの検証の鋭さ
強みテストの真価を暴く
弱み実行コストが高い

ミューテーションは「コードを意図的に壊して、テストが気づくか」を見る。気づければ「殺害」、気づかなければ「生存」。

// なぜ生まれたか 問題: カバレッジは「実行したか」しか見ない 実行されたが検証していない罠 ↓ 発想の転換: 「実行範囲」を測るのをやめて 「壊した時に気づけるか」を測ろう ↓ 仕組み: コードを自動で少しずつ変える テストを走らせる テストが落ちれば 「気づいた」 通ってしまえば 「気づかない」 ↓ 結果: 生存変異 = テストの穴 気づかない場所が分かる ↓ 「量」から「質」へ 指標の発想が次世代に進化
この進化の意味
カバレッジ→ミューテーションの流れは、「測りやすいもの」から「本質的なもの」へと指標が進化していく一例。完全ではないが、より核心に近づこうとする試みです。だがミューテーション100%もまた目標化すれば壊れる。グッドハートの法則は変わらず効くのです。
07

測定の上手な付き合い方

HEALTHY MEASUREMENT

ここまでの構造を踏まえ、測定とどう付き合えばよいかをまとめます。指標は使うが、振り回されない。数字を見るが、本質を見失わない。バランス感覚の指針を持っておくと、現場でぶれない判断ができます。

healthy use 指標と健全に付き合う原則
// 5つの原則
  1. 指標は代理であり、本質ではないと意識する
  2. 指標は下を見る道具として使う（弱点発見）
  3. 指標を目標化しない（グッドハートの罠）
  4. 複数の指標を組み合わせて見る（量+質）
  5. 最後は人の判断が必要と認める
// 指標との健全な距離感 良い使い方: ・「全体カバレッジ70%か」と観察する ・「payment.jsが30%」と弱点を見つける ・「下がってきた」と傾向を捉える ・複数指標を合わせ技で見る 悪い使い方: ・「80%強制」と目標にする ・「100%でないと却下」と絶対視する ・1つの指標だけで判断する ・指標を評価指標に使う ↓ 指標は鏡である 磨くべきは映る対象 鏡そのものではない // 数字に振り回されず // 数字を道具として使う // この距離感が現場の知恵
この巻のまとめ
本編Vol.8で学んだカバレッジ・ミューテーションは、テストの「量と質」を測る道具でした。地図編Vol.8では、これらが「代理指標」であり、「目標化すると壊れる」性質を持つことを構造的に理解しました。数字を見るが、本質を見失わない。この姿勢を持って次の章へ進みましょう。
