01

学習者がたどる4段階

FOUR STAGES

テストエンジニアリングの学習は、4つの段階を経て深まっていきます。それぞれの段階で見える景色が違い、つまずく場所も違います。自分が今どこにいるかを意識すると、次に学ぶべきものが見えてきます。

4 stages 学習者がたどる成長段階
// 4つの段階
LV.1
書けるテストを1つ書いて緑にできる。AAAパターンに沿って書ける。本編Vol.1〜2の世界
LV.2
続けられる業務で日常的にテストを書く。フレーキーやモックの判断ができる。本編Vol.3〜5の世界
LV.3
使いこなせる結合・E2Eまで含めて層を組める。カバレッジを観察対象として使える。本編Vol.6〜8の世界
LV.4
戦略を立てられるチームとしてテストを設計し、CI/CDで継続的に運用できる。本編Vol.9〜10の世界
// 各段階での到達点 LV.1 書ける 1つのテストを書いて緑にできる ↓ これだけでも価値は十分 LV.2 続けられる 業務でテストを書く習慣 ↓ 壁: モックとフレーキー LV.3 使いこなせる ピラミッド全層を扱う ↓ 壁: 何を測り、何に集中するか LV.4 戦略を立てられる チーム・組織レベル ↓ 壁: 文化・習慣の変革 ↓ 焦らず一段ずつ 下を固めてから次へ // LV.1で止まっても価値あり // LV.4は数年かけて到達する地点
POINT
一足飛びは難しい、というのが現実。LV.1とLV.2の間で挫折する人が一番多い。最初の3ヶ月を乗り越えると、テストが習慣になります。本シリーズはどの段階の読者にも、その段階で必要な視座を提供することを意図しました。

03

地図編で使った3つの軸

THREE AXES

地図編9巻を通して、本編の内容を読み解くのに3つの軸を繰り返し使ってきました。時間軸・上下軸・左右軸。これらは技術書を読む時の汎用ツールでもあります。シリーズを離れた後も、別の技術領域で使える視座です。

three axes 地図編が用いてきた3つの視点
// 3つの軸とその使い方
TIME — 時間軸 歴史と因果を辿る視点。「なぜそうなったか」を時代の流れで読む
VERTICAL — 上下軸 階層・抽象度の視点。「全体と部分」「上位と下位」の関係を見る
HORIZONTAL — 左右軸 対比・対立の視点。「内側と外側」「純粋と不純」を並べて見る

地図編Vol.1, 3, 9 = 主に時間軸
地図編Vol.2, 6 = 主に上下軸
地図編Vol.3, 4, 7 = 主に左右軸
地図編Vol.5, 8 = 構造軸（複合）

// 軸を使い分ける思考法 何かを理解しようとした時: 「なぜ今こうなっているのか?」 → 時間軸を使う → 歴史を辿る 「全体のどこに位置するのか?」 → 上下軸を使う → 階層で整理 「何と何が違うのか?」 → 左右軸を使う → 対比して並べる ↓ これは技術書全般に効く 汎用的な読み解きツール // シリーズを離れても // 別の領域で使える視点 // 「3軸で読む」を意識するだけで // 学習効率が大きく変わる
この巻からの贈り物
地図編が伝えたかった最大のメッセージは「点を線で・線を面で読み解く」姿勢。3つの軸はそのための道具です。技術を学ぶ時に「点」を集めるだけでなく、それらを「軸の上に並べて構造を見る」習慣が、知識を深くします。
04

ピラミッドを3軸で読む

PYRAMID IN 3 AXES

地図編で繰り返し登場したテストピラミッドは、3つの軸すべてで読み解けます。同じ概念が、軸を変えるたびに違う顔を見せる—これが「知識の立体化」の典型例です。

pyramid x 3 ピラミッドを3軸で読む
// ピラミッドの3つの読み方
粒度の上下軸
(MAP Vol.2) 単体→結合→E2Eの粒度違い。下が広く上が狭い理由は「速度×数」の経済原理
構造の上下軸
(MAP Vol.6) 関数→クラス→モジュール→システムの組み上がり。ソフト構造そのものがピラミッド型だから
視点の左右軸
(MAP Vol.7) 内側視点(下層)と外側視点(上層)。同じシステムを別の角度から見る
// 3軸で見ると見える「同じこと」 粒度: 単体 多・速・安定 E2E 少・遅・脆い → ピラミッド型が経済的 構造: 関数 数千 システム 1 → ピラミッド型が必然 視点: 内側 = 下層 (部品ごと) 外側 = 上層 (全体) → ピラミッド型が対応 ↓ 3つの違う角度から 同じ結論にたどり着く // これが「知識が立体化する」体験 // テストピラミッドが // 「ただの図」から // 「複層的な真理」に育つ
この立体化の意味
本編で1度学んだ「テストピラミッド」が、地図編Vol.2・Vol.6・Vol.7で3回違う角度から再訪されました。同じ概念に複数回違う光を当てることで、知識は1次元の事実から、3次元の理解へと変わります。これが「地図編」というコンセプトの狙いでした。

シリーズ全体の振り返り

RETROSPECTIVE

最後に、本シリーズが伝えたかったことを3つの核心メッセージに絞ってまとめます。技術の細部はいずれ忘れても、この3点だけは持ち帰っていただきたい。これがシリーズの本当の到達点です。

three messages 本シリーズの3つの核心
// 3つのメッセージ
ONE
テストは将来の自分を救う書いた瞬間ではなく、半年後、1年後、新人が触る時、リファクタする時、本物の価値が現れる
TWO
ピラミッド構造を意識する下を厚く、上を薄く。粒度・構造・視点、3軸で見ると揺らがない原則
THREE
仕組みで品質を作る意志や規律ではなく、CI/CDという仕組みで継続的に検証する。これが現代の到達点
// シリーズ全体を一文で テストとは: 「現在の安心」のためではなく 「未来の自由」のために 書くもの ↓ 学んだ全ては: ・AAAパターン ・TDD ・テストダブル ・設計技法 ・結合とE2E ・カバレッジ ・CI/CD ・トラブル対処 すべて「未来の自由」を作る道具 ↓ 最終的に: あなたのコードが 半年後・1年後にも 変更しやすく 恐れずに改善できる状態 ↓ それこそが エンジニアリングテストの到達点
