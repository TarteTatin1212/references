Grid 基礎編

CSS GRID FUNDAMENTALS — BLUEPRINT SERIES

Flexboxが「1方向の並び」なら、Gridは「行と列の2次元を同時に操る」仕組みです。 grid-template-columnsで列の数と幅を宣言するだけで、子要素が自動的に格子状に並びます。 fr単位とrepeat()の2つを覚えるだけで、 複雑なページレイアウトが驚くほど少ないコードで完成します。

01

FlexとGridの使い分け

FLEX VS GRID

「Flexがあれば十分では？」——いいえ、得意な場面が違います。 Flexは「コンテンツの量に合わせて柔軟に並べる1次元」、 Gridは「先にマス目を宣言して配置する2次元」です。どちらが優れているのではなく、目的で選びます。

flex vs grid 1次元と2次元、コンテンツ起点とレイアウト起点の違い

/* Flexbox → コンテンツが主役 */
/* 「並べたいものがある → 並べる」*/
.nav {
  display: flex;
  gap: 16px;
  /* リンクが増減しても自動で調整 */
}

/* Grid → レイアウトが主役 */
/* 「先にマス目を作る → 配置する」*/
.page {
  display: grid;
  grid-template-columns: 240px 1fr;
  grid-template-rows: 60px 1fr 60px;
  /* 先に設計図を書く */
}

  1方向（横か縦）→ Flex
  2方向（行と列）→ Grid
  ページの骨格  → Grid
  コンポーネント内部 → Flex が多い
  
  両方使う→ Grid でページを作り
            中身の要素は Flex で並べる
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━ */

display: gridを書いただけでは見た目は変わりません。 Flexと違い、Gridは列（または行）の定義がセットで必要です。 grid-template-columnsで列を宣言した瞬間に格子が生まれます。

03

grid-template-columns — 列の定義

GRID TEMPLATE COLUMNS

grid-template-columnsは列の数と各列の幅を宣言します。 値を半角スペースで区切って並べると、その数だけ列が生まれます。 px・%・fr・auto など複数の単位を混在させることも可能です。

grid-template-columns スペース区切りで列を宣言 / 値の数 = 列数

/* 列の幅の指定方法 */

/* 固定幅 */
.grid {
  grid-template-columns: 200px 200px 200px;
}

/* サイドバー固定 + メイン可変 */
.grid {
  grid-template-columns: 240px 1fr;
}

/* 3列：左右サイドバー + メイン */
.grid {
  grid-template-columns: 240px 1fr 200px;
}

/* 比率で分割（2:1:1）*/
.grid {
  grid-template-columns: 2fr 1fr 1fr;
}

/* %, minmax, autoも使える */
.grid {
  grid-template-columns:
    minmax(200px, 1fr) 1fr;
  /* 最小200px・最大1frの列 */
}

/* ↑ 値の数 = 列の数
   スペースで区切るだけ！ */

fr単位 — 余白を分け合う比率

FR UNIT

fr（fraction＝分数）はコンテナの余白を比率で分配するGrid専用の単位です。 Flexboxのflex-growに似た発想ですが、Gridではより直感的に列の比率を指定できます。 固定幅（pxや%）と組み合わせると「固定＋可変」レイアウトが簡単に作れます。

fr unit Grid専用の「余白分配」単位 / px・%と混在可
// fr の分配の仕組み（視覚化）
1fr 1fr 1fr → 残り幅を3等分

1fr
33%
1fr 2fr → 1:2の比率に分割

2fr（67%）
60px 1fr → px固定後の残りをfrで取る

固定
残り全部を1fr
80px 1fr 1fr 80px → 外側固定・内側均等
80px
1fr
1fr
80px
// px分を引いた残りをfrで按分する 
/* fr = fraction（残余領域の分数）*/

/* 全体を3等分 */
.grid {
  grid-template-columns: 1fr 1fr 1fr;
}

/* 左1/3、右2/3 */
.grid {
  grid-template-columns: 1fr 2fr;
}

/* サイドバー固定 + メイン可変 ★ */
.grid {
  grid-template-columns: 240px 1fr;
  /* 240px を除いた残りを1frが取る */
}

/* % との違い */
/* % → コンテナ全体に対する割合   */
/* fr → gap除いた残余領域の割合   */

/* gapがある時 frが自然に正確 ★ */
.grid {
  display: grid;
  grid-template-columns: 1fr 1fr 1fr;
  gap: 16px;
  /* % だと gap 分ずれる…
     fr なら gap を自動的に除外 */
}
POINT — frの最大の利点はgapを自動的に除外して按分することです。 %でカラムを作るとgapの分だけ計算が狂いますが、frはgapを引いた後の残余を分配するので常に正確です。Gridで列幅を指定するなら基本はfrを使います。

repeat() — 繰り返しを簡潔に書く

REPEAT FUNCTION

同じ列定義を何度も書くのは冗長です。repeat(回数, 幅)を使うと繰り返しをコンパクトに書けます。 さらにauto-fillやauto-fitと組み合わせると、 列数をブラウザが自動計算するレスポンシブグリッドが1行で完成します（Vol.5で詳しく扱います）。

  grid-template-columns: repeat(12, 1fr);

  /* 240px + 均等3列 */

.cards {
  display: grid;
  grid-template-columns:
    repeat(auto-fill, minmax(280px, 1fr));
  gap: 20px;
}
/* ↑ 280px以上あれば並べ、

      → メディアクエリ不要！  */

grid-template-rows — 行の定義

GRID TEMPLATE ROWS

行の高さはgrid-template-rowsで定義します。 省略するとコンテンツの高さに合わせた暗黙的な行が自動生成されます。 ページレイアウトではヘッダー・フッターを固定高さにし、メインを1frで可変にするのが定番です。

単位意味

1fr残りを比率で
autoコンテンツに合わせる
minmax(100px,1fr)最小100px、最大1fr
min-contentコンテンツの最小幅
/* grid-template-rows */

/* 省略可：コンテンツ高さに自動調整 */
.grid {
  display: grid;
  grid-template-columns: 1fr 1fr;
  /* rows 省略 → 行は自動生成 */
}

/* ページ構造には明示する */
.page {
  display:             grid;
  grid-template-rows:  60px 1fr 60px;
  min-height:          100vh;
  /* ↑ ヘッダー60px / メイン可変 / フッター60px
        min-height:100vh でフッターが必ず下へ */
}

/* shorthand: grid */
.page {
  display: grid;
  grid: 60px 1fr 60px / 240px 1fr;
  /* rows / columns をまとめて */
}

/* 暗黙的な行の高さを制御 */
.grid {
  grid-auto-rows: 200px;
  /* 自動生成される行を全て200px */
}
.grid {
  grid-auto-rows: minmax(100px, auto);
  /* 最低100px、コンテンツで伸びる */
}

gap — グリッドの溝

GAP

GridのgapはFlexと同じプロパティです。 グリッドの場合、行間（row-gap）と列間（column-gap）を別々に設定できるため、 カードのような縦横異なる余白にも簡単に対応できます。

.grid {
  display: grid;
  grid-template-columns: repeat(3, 1fr);

  /* 縦横同じ */
  gap: 20px;

  /* 縦 / 横 を別々に */
  gap: 32px 16px; /* row col */

  /* 個別指定 */
  row-gap:    32px;
  column-gap: 16px;
}

/* カードグリッドの定番 */
.card-grid {
  display: grid;
  grid-template-columns:
    repeat(3, 1fr);
  row-gap:    32px; /* 行間は広め */
  column-gap: 20px; /* 列間は狭め */
}

/* ━━ frとgapの計算 ━━━━━━━━━━━━━
  コンテナ幅900px、gap:20px、3列の場合
  
  使えるfr幅 = 900 - (20 × 2) = 860px
  1fr = 860 ÷ 3 ≒ 286.7px
  
  % で指定すると gap 分だけずれる
  → fr を使う理由がここにある
━━━━━━━━━━━━━━━━━━━━━━━━━━━━ */

実践：ページレイアウトを組む

PAGE LAYOUT PATTERN

「ヘッダー・サイドバー・メインコンテンツ・フッター」というWebサイトの定番骨格を Gridで組み立てます。コードはわずか10行。 これがGridが「ページレイアウト向き」と言われる理由です。

holy-grail-layout header / sidebar / main / footer を10行で
// 実際に動くページレイアウト
⊞ HEADER — 全幅（grid-column: 1 / -1）
SIDEBAR
ナビ 01
ナビ 02
ナビ 03
MAIN CONTENT
FOOTER — 全幅（grid-column: 1 / -1）
// header と footer が2列を横断している
構造のポイント
columns: 160px 1fr
rows:    44px 1fr 36px
header: grid-column 1 / -1 → 全幅
footer: grid-column 1 / -1 → 全幅 
/* ── 定番ページレイアウト ── */
.page {
  display: grid;
  grid-template-columns: 240px 1fr;
  grid-template-rows:    60px 1fr 60px;
  min-height: 100vh;
  gap: 0;
}

/* ヘッダー：全幅（1列目〜最後の列）*/
.header {
  grid-column: 1 / -1;
  /* -1 = 最後のライン */
}

/* サイドバー：自動配置でOK */
.sidebar { }

/* メイン：自動配置でOK */
.main { }

/* フッター：全幅 */
.footer {
  grid-column: 1 / -1;
}

/* HTML構造 */
/* <div class="page">          */
/*   <header>...</header>     */
/*   <nav>サイドバー</nav>    */
/*   <main>メイン</main>      */
/*   <footer>...</footer>    */
/* </div>                     */
POINT — grid-column: 1 / -1の-1は「最後の列ライン」を意味します。列の数に関係なく「全幅にまたがる」という指定ができるので、列数を変えてもヘッダーとフッターのコードを変える必要がありません。Grid特有の超便利な書き方です。列ラインの指定については Vol.5 で詳しく扱います。
