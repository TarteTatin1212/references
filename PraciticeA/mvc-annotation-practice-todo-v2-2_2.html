03

📚 必要な前提知識

PREREQUISITES

DBに不慣れな方のために、最小限の前提を整理しておきます。「DBが何をするものか」「JDBCとは何か」「MyBatisとは何か」が分かれば本巻は十分です。

key-concepts 本巻で出てくる用語
// 用語の整理
TERM 1
データベース（DB）
アプリのデータを保存しておく場所。MySQLはその管理ソフト。アプリが終了してもデータは残る。
TERM 2
テーブル
DBの中の「表」。Excelの1シートのイメージ。列（カラム）と行（レコード）からなる。
TERM 3
DDL ／ DML
DDL=テーブル定義のSQL（CREATE TABLE等）／DML=データ操作のSQL（SELECT/INSERT等）。
TERM 4
JDBC
「JavaがDBと話すための共通言語」。MySQLでもPostgreSQLでも、同じJavaコードで操作できるようにする仕組み。
TERM 5
MyBatis
「SQLとJavaオブジェクトを橋渡し」するライブラリ。SQLを書くだけで、結果セットを自動でJavaクラスに詰めてくれる。Vol.4で詳しく扱う。
// アプリとDBの関係図 ┌─────────────────┐ │ Spring Boot App │ │ （Javaコード） │ └──────┬──────┘ │ MyBatis │ ↓ │ JDBC（共通言語） │ ↓ │ MySQL Driver（翻訳機） ▼ ┌─────────────────┐ │ MySQL Server │ │ ┌──────────┐ │ │ │ todo_db │ │ │ │ └ tasks │ │ │ └──────────┘ │ └─────────────────┘ // ───────────────────── // この絵を頭に入れておくと、 // 「どこで何のエラーか」が // 切り分けやすくなる。 // Vol.2で扱う最小のMyBatis // 動作確認用なので「最小構成」： // 1. Mapperインターフェイス @Mapper public interface RawTaskMapper { @Select("SELECT * FROM tasks") List<Map<String, Object>> findAllRaw(); } // 2. それを呼ぶController @RestController public class RawTaskController { private final RawTaskMapper mapper; ... } // 結果は Map で受けます（簡易）。 // Vol.3で「ちゃんとした Task クラス」を // 作って、Map を Task に置き換えます。
POINT ──

v2のVol.2では「Mapを使った仮の実装」でDBから画面までを一気に繋ぎます。Vol.3で正式なTaskクラスを作ったら、このMapを置き換えます。「動くものを早く見たい」を最優先にしたv2ならではの構成です。

🔌 MySQLに接続する

CONNECT TO MYSQL

まずはMySQLが起動しているか確認し、コマンドラインから接続できる状態にします。「アプリの前に手動で接続を確認する」のは業務でも超大事な手順です。

mysql-connect MySQLへのログイン確認
// 接続確認の手順
サービス起動確認（Windows）
  1. Win + R → services.msc
  2. 「MySQL80」（または「MySQL84」「MySQL90」等）のサービスを探す
  3. 状態が「実行中」になっているか確認
  4. 止まっていたら右クリック→「開始」

  • 場所は通常 C:\Program Files\MySQL\MySQL Server 8.0\bin

05

📐 DBとテーブルを作る

CREATE DATABASE ＆ TABLE

MySQLにログインできたら、todo_dbとtasksテーブルを作成し、動作確認用のサンプルデータを投入します。v2では3件入れて、Vol.2のラストでこの3件が画面に出ることを目指します。

tasks-schema テーブル設計の意図
// 各列の役割
列名型役割
id BIGINT 主キー。自動連番（AUTO_INCREMENT）
title VARCHAR(100) タスクのタイトル。最大100文字
done BOOLEAN 完了フラグ。デフォルト FALSE
created_at DATETIME 作成日時。デフォルトで現在時刻

設計のポイント：

  • idは必ず主キーにする ── データを一意に特定するため
  • BIGINTを選ぶ ── INT(約21億)より大きく、将来安心
  • created_atを持つ ── 作成順での並び替えが可能になる
  • utf8mb4を使う ── 絵文字も格納できる
合言葉：テーブル設計は「シンプルに始めて、必要になったら追加」。最初から完璧を目指さない。
// 順番に実行 // ── ① DBを作成 ── mysql> CREATE DATABASE todo_db -> CHARACTER SET utf8mb4 -> COLLATE utf8mb4_unicode_ci; Query OK, 1 row affected // ── ② 使うDBを切り替え ── mysql> USE todo_db; Database changed // ── ③ テーブルを作成 ── mysql> CREATE TABLE tasks ( -> id BIGINT AUTO_INCREMENT PRIMARY KEY, -> title VARCHAR(100) NOT NULL, -> done BOOLEAN NOT NULL DEFAULT FALSE, -> created_at DATETIME NOT NULL -> DEFAULT CURRENT_TIMESTAMP -> ); Query OK, 0 rows affected // ── ④ 確認 ── mysql> DESC tasks; +------------+--------------+------+-----+ | Field | Type | Null | Key | +------------+--------------+------+-----+ | id | bigint | NO | PRI | | title | varchar(100) | NO | | | done | tinyint(1) | NO | | | created_at | datetime | NO | | +------------+--------------+------+-----+ // ── ⑤ サンプルデータ（3件） ── mysql> INSERT INTO tasks (title, done) VALUES -> ('最初のタスク', false), -> ('Spring Bootを学ぶ', false), -> ('牛乳を買う', true); Query OK, 3 rows affected // ── ⑥ 確認 ── mysql> SELECT * FROM tasks; +----+-------------------+------+-----------+ | id | title | done | created..| +----+-------------------+------+-----------+ | 1 | 最初のタスク | 0 | 2026-... | | 2 | Spring Bootを学ぶ | 0 | 2026-... | | 3 | 牛乳を買う | 1 | 2026-... | +----+-------------------+------+-----------+ 3 rows in set // ───────────────────── // これでDB側の準備は完了。 // 次は Spring Boot から繋ぎます。
POINT ──

BOOLEANはMySQLでは内部的にTINYINT(1)に変換されます。表示は0/1ですが、論理的にはfalse/true。MyBatisを通すとJavaのBoolean型に自動変換されるので意識不要です。

DBは作りました。次はSpring Bootに「ここにDBがあるよ」と教える必要があります。Vol.1の起動エラーの原因はここの設定が無いこと。本セクションを完了すればVol.1のエラーが自動で解消します。

設定項目の意味：

キー意味

mybatis.configuration.

JDBC URLの重要パラメータ：

  • useSSL=false ── 開発時はSSL不要
  • allowPublicKeyRetrieval=true ── MySQL 8の認証で必須
  • characterEncoding=UTF-8 ── 日本語の文字化け防止
重要：MySQL 8 デフォルトの認証プラグインcaching_sha2_passwordでuseSSL=falseのときはallowPublicKeyRetrieval=trueが必須。これを忘れると「Public Key Retrieval is not allowed」エラー。
// application.properties（完成版） # ───────────────────────── # Application 基本設定 # ───────────────────────── spring.application.name=todo # ───────────────────────── # データソース（MySQL接続） # driver-class-name は省略可 # （Spring Bootが URL から自動判定） # ───────────────────────── spring.datasource.url=jdbc:mysql://localhost:3306/todo_db?useSSL=false&allowPublicKeyRetrieval=true&characterEncoding=UTF-8 spring.datasource.username=root spring.datasource.password={各自のパスワード} # ───────────────────────── # MyBatis 設定 # created_at → createdAt に自動変換 # ───────────────────────── mybatis.configuration.map-underscore-to-camel-case=true # ───────────────────────── # ログ（SQLが見えると便利） # Vol.4以降のデバッグで活躍 # ───────────────────────── logging.level.com.example.todo=DEBUG # ───────────────────────── # 補足：DevToolsが入っているので # Thymeleafキャッシュは自動でfalse。 # 明示しなくてもOK。 # ───────────────────────── // 重要な注意点：URLは1行で書く // ✗ 実は properties ファイルでは // バックスラッシュで改行できますが、 // 本書では分かりやすさ重視で // 1行にまとめます。 // ✓ こう書く（実際は折り返さない）： spring.datasource.url=jdbc:mysql://localhost:3306/todo_db?useSSL=false&allowPublicKeyRetrieval=true&characterEncoding=UTF-8 // 設定後の動作（重要） // DevToolsが入っているので、 // application.properties を保存すると： INFO --- Restarting due to change INFO --- HikariPool-1 - Starting... INFO --- HikariPool-1 - Start completed. // ★ INFO --- Started TodoApplication in 0.8s // HikariPool が起動できれば DB 接続成功！ // Vol.1の起動エラーは自動で解消します。
裏技 ──

logging.level.com.example.todo=DEBUGは超便利。MyBatisが発行したSQLがコンソールに表示されるようになります。Vol.4以降の動作確認で絶大な効果を発揮します。本書では最初から有効化しておきましょう。

raw-package 動作確認用パッケージとファイル
// クラス作成手順
パッケージを作る

  2. New → Package
  3. Name欄にcom.example.todo.rawと入力

  1. rawパッケージを右クリック

注目：@Mapper付けたインターフェイスはMyBatisが自動で実装を生成します。これがMyBatisの神髄。詳しくはVol.4で。
// RawTaskMapper.java（動作確認用） package com.example.todo.raw; // ↓ import文は省略せず全部書きます import java.util.List; import java.util.Map; import org.apache.ibatis.annotations.Mapper; import org.apache.ibatis.annotations.Select; @Mapper public interface RawTaskMapper { @Select("SELECT * FROM tasks ORDER BY id") List<Map<String, Object>> findAllRaw(); } // ───────────────────── // 結果を Map で受けています。 // // 通常は List<Task> で受けますが、 // Vol.3でTaskクラスを作るまでは // Map で代用する作戦です。 // // 一時的な動作確認用なので OK。 // RawTaskController.java（動作確認用） package com.example.todo.raw; import java.util.List; import java.util.Map; import org.springframework.web.bind .annotation.GetMapping; import org.springframework.web.bind .annotation.RestController; @RestController public class RawTaskController { private final RawTaskMapper mapper; // コンストラクタ注入 public RawTaskController(RawTaskMapper mapper) { this.mapper = mapper; } @GetMapping("/raw") public List<Map<String, Object>> rawList() { return mapper.findAllRaw(); } } // ───────────────────── // たった10行ちょっとで完成。 // /raw にアクセスするとDBの中身が // JSONで返ってきます。 // 補足：コンストラクタ注入とは？ // Spring が裏で： // 1. RawTaskMapper の実装を MyBatis が生成 // 2. それを Bean として登録 // 3. RawTaskController を作る時、 // コンストラクタの引数に // 自動でその Bean をセット // // → 自分で new しないで OK // // 詳しくは Vol.5 で。
POINT ──

Mapperはインターフェイスであることに注目。Eclipseで「New → Interface」を選んでください（Classじゃない）。「SQLという仕事を宣言するだけ。実装はMyBatisが自動生成」── Vol.1のSection 06で扱った話の実例がここで登場します。

🚨 ハマりどころと対処

COMMON PITFALLS

DB接続トラブルは初学者にとって最大の難関。エラーの種類を見れば原因はほぼ分かるので、典型パターンを覚えておきましょう。「エラーメッセージは敵じゃなく地図」です。

db-troubles DB接続でハマる5パターン
// よくあるトラブル
PITFALL 1
Public Key Retrieval is not allowed
MySQL 8 認証プラグインの問題。URL にallowPublicKeyRetrieval=trueを付け忘れ。Section 6 で示した完全な URL を使うこと。
PITFALL 2
Access denied for user
パスワード間違い。application.propertiesのspring.datasource.passwordを確認。コピペ時の余分なスペース注意。
PITFALL 3
Communications link failure
MySQLが起動していない。サービスを確認。services.mscで「MySQL80」を開始。
PITFALL 4
Unknown database 'todo_db'
DBがまだ作られていない。Section 5 のCREATE DATABASEを実行したか確認。
PITFALL 5
No MyBatis mapper was found
Mapperインターフェイスに@Mapperを付け忘れ、またはcom.example.todoの下のパッケージに置いていない。階層を確認。
合言葉：エラーメッセージの最初の数行を「読み飛ばさず」読む。たいてい原因が書いてある。

DB接続トラブルは「アプリの問題か、DBの問題か」を切り分けるのが第一歩。コマンドラインで接続できればDBは正常 → アプリ側の設定ミス。コマンドラインでも繋がらなければDB側の問題。これを切り分ける癖が業務で役立ちます。
