PART 1 — 列挙型 enum
01

列挙型（enum）の基本

ENUM BASICS

enum（イーナム／列挙型）は、「選択肢が決まっている値」を表す専用クラスです。信号の色（赤・黄・青）、曜日（月〜日）、方角（東西南北）など。「この中のどれか」しかありえない値を、安全に扱えます。

enum 決まった選択肢を表す
// たとえ：信号の色は3つだけ
RED YELLOW GREEN

信号に「紫」や「水玉」はありません。この3つしかないと決まっている。enum はそれをコードで表現します。

// enum の定義 enum Signal { RED, YELLOW, GREEN } // この3つ以外の Signal は作れない // もし文字列で管理すると… String s = "あか"; // ❌ タイプミスし放題 // enum なら間違った値が入らず安全！
02

enumの使い方

USING ENUM

enumの値は Signal.RED のように「型名.値」で取り出します。switch と組み合わせると、「どの選択肢か」で処理を分けるのがとてもキレイに書けます。これがenumの定番の使い方です。

enum + switch 選択肢ごとに処理を分ける
// 動かすと…

Signal.RED を渡すと「止まれ」、GREEN なら「進め」。選択肢ごとの分岐が読みやすく書けます。

出力（RED を渡した場合）
止まれ
// enum で分岐 Signal sig = Signal.RED; switch (sig) { case RED -> System.out.println("止まれ"); case YELLOW -> System.out.println("注意"); case GREEN -> System.out.println("進め"); } // case に存在しない値を書けない＝安全
POINT
enumの一番のメリットは「ありえない値が入らない」こと。文字列や数字で管理すると、タイプミスや想定外の値でバグります。選択肢が決まっているなら迷わずenum、が鉄則です。
03

enumに値を持たせる（応用）

ENUM WITH FIELDS

enumは実は各選択肢に追加情報を持たせられます。たとえば惑星に「重力」を、信号に「待ち時間」を。ここはやや応用なので「こんなこともできる」程度でOKです。

enum field 各値にデータを紐づける
// 各色に秒数を持たせる

RED は30秒、GREEN は25秒…のように、選択肢ごとに固有の値を持たせられます。

単なる「ラベル」を超えて、データ付きの選択肢として使えるのがenumの底力です。

// 各値に秒数を持たせる enum Signal { RED(30), YELLOW(5), GREEN(25); final int seconds; Signal(int s) { // enum用コンストラクタ seconds = s; } } // 使う時 System.out.println(Signal.RED.seconds); // → 30
PART 2 — アノテーション annotation
04

アノテーションの基本

ANNOTATION BASICS

アノテーションは @ で始まる「コードに貼る付箋・ラベル」です。実は @Override をこれまで何度も使ってきましたね。あれがアノテーションです。プログラムの動きを変えるのではなく、「目印」を付けるのが役割です。

@Annotation コードに付ける目印
// たとえ：本に貼る付箋

本の重要ページに付箋を貼っても、本の中身は変わりません。でも「ここ大事」という情報が増えます。

アノテーションも同じ。コードに「これはこういう意味ですよ」という情報を添える道具です。コンパイラやツールがそれを読み取ります。

// もう使っていた @Override class Dog extends Animal { @Override // ← これがアノテーション void cry() { System.out.println("ワン"); } } // 「これは親メソッドの上書きですよ」 // という目印。書き間違えると警告が出る
05

よく使うアノテーション

COMMON ANNOTATIONS

初学者がまず出会う定番アノテーションを押さえましょう。自分で作ることもできますが（だから「Annotation Interfaces」と呼ばれます）、最初は「使う側」として知っていれば十分です。

common ones まず覚える3つ
// 定番3種
  • @Override：親メソッドの上書き宣言
  • @Deprecated：「もう古い、使わないで」の印
  • @SuppressWarnings：警告を黙らせる

特に @Override は付ける習慣をつけると安全です。タイプミスを防いでくれます。

// 定番アノテーションの例 @Override void run() { ... } @Deprecated // 「この機能は古い」 void oldMethod() { ... } @SuppressWarnings("unchecked") // 警告を抑制 void tricky() { ... }
TIP
現場のフレームワーク（Spring など）では @Controller や @Service のような独自アノテーションが大量に登場します。「@で始まる付箋なんだな」と分かっていれば、初見でも怖くありません。
PART 3 — レコードクラス record
06

レコードクラス（record）の基本

RECORD BASICS

record（レコード）は「データを入れる箱」を一行で作れる新しいクラスです。名前と年齢、X座標とY座標——こういう「データをまとめて持つだけ」のクラスを、驚くほど短く書けます。

record データの箱を一行で
// 普通のクラスだと…

「名前と年齢を持つだけ」のクラスでも、普通に書くとフィールド・コンストラクタ・getter…と何十行になります。

record ならたった一行。面倒な部分を全部自動生成してくれます。

// 普通のクラス（長い…） class Point { private int x; private int y; Point(int x, int y) { this.x = x; this.y = y; } int x() { return x; } int y() { return y; } // equals, hashCode, toString も… } // record なら一行！ record Point(int x, int y) {}
07

recordのうれしさ

WHY RECORD

recordは一行書くだけで、必要なものを全部用意してくれます。コンストラクタ、値を取り出すメソッド、表示用の文字列、比較機能——全部自動。「データの箱」を作るならrecordが圧倒的にラクです。

auto-generated 面倒を全部自動でやってくれる
// 自動で付いてくるもの
  • コンストラクタ（値を受け取る）
  • 値を取り出すメソッド x() y()
  • 見やすい toString()
  • 中身が同じか比べる機能

これを手で書くと数十行。それが一行で済むのです。

// record はすぐ使える record Point(int x, int y) {} Point p = new Point(3, 5); System.out.println(p.x()); // → 3 System.out.println(p); // → Point[x=3, y=5] // 全部自動。データの箱はこれでOK
POINT
これだけ覚えれば初心者卒業：enum＝決まった選択肢、annotation＝コードへの付箋、record＝データの箱。3つとも「面倒を減らす道具」。深い使い方は必要になった時に辞書として開けばOKです。
