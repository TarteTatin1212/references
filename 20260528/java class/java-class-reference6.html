LOCAL & ANONYMOUS CLASS — disposable classes / chapter 9-10

塊③の続きです。今回は「その場で使い捨てるクラス」2種類。ローカルクラスはメソッドの中だけで使うクラス。匿名クラスはなんと名前すら付けないクラスです。「名前がないクラスって意味あるの？」——あります。むしろ現場では匿名クラスのほうが頻繁に登場します。実例で慣れていきましょう。

ローカルクラス 使える範囲 いつ使うか 匿名クラス 書き方 現場での使われ方
01

ローカルクラス（Local Class）

LOCAL CLASS

メソッドの中に書くクラスがローカルクラスです。ローカル変数（メソッド内の変数）と同じ感覚。そのメソッドの中でしか使えません。メソッドを抜けたら消えてしまう、限定的なクラスです。

local class メソッドの中だけのクラス
// たとえ話：会議室のホワイトボード

会議室のホワイトボードに書いたメモを想像してください。その会議の間だけ使い、終わったら消す。外には持ち出しません。

ローカルクラスも、そのメソッドの中だけの一時的なクラスです。

// メソッド内にクラスを定義 class Greeter { void run() { // メソッドの中でクラスを宣言！ class Message { void show() { System.out.println("やあ"); } } // この中でだけ使える Message m = new Message(); m.show(); } }
02

使える範囲（スコープ）

SCOPE

ローカルクラスは宣言したメソッドの中だけに存在します。メソッドの外からは見えません。だから「他で使う予定が全くない、その場限りのクラス」にぴったりです。

scope.vis 見える範囲はメソッドの中だけ
// 見える範囲
method run() {
class Message ← ここでだけ見える
}
↑ メソッドの外からは Message は見えない
// 範囲のルール void run() { class Message {} new Message(); // ⭕ ここはOK } void other() { new Message(); // ❌ 別メソッドからは見えない }
03

いつ使うか

WHEN TO USE

正直に言うと、ローカルクラスは出番がかなり少ないです。「1つのメソッドの中だけで、何度か使う複雑な処理を、クラスにまとめたい」という限定的な場面で使います。読めればOKのテーマです。

rare use 出番は少なめ、読めれば十分
// 結論
ローカルクラスの立ち位置
「メソッド内専用のクラス」。でも実務では次の匿名クラスのほうが圧倒的によく使われる。

なので、ここはサラッと「こういうのもある」で先へ進みましょう。本命は次のセクションです。

// ローカルクラスのまとめ // ・メソッドの中に書くクラス // ・そのメソッド内でだけ使える // ・出番は少なめ // ・「読めればOK」でOK // → より実用的な「匿名クラス」へ続く
04

匿名クラス（Anonymous Class）

ANONYMOUS CLASS

ここからが本命。名前を付けないクラス、それが匿名クラスです。「クラスを定義して、すぐ1回だけ new する」を一気にまとめて書く仕組み。インターフェースや抽象クラスをその場で実装したい時に大活躍します。

anonymous class 名前なし・その場で1回きり
// なぜ名前がいらない？

一度しか使わないクラスに、わざわざ名前を付けるのは面倒。だったら名前を省略して、その場で中身を書いて即 new しよう——という発想です。

使い捨ての紙コップに名前を書かないのと同じ。一回きりだから名前は不要なのです。

// まず「普通のやり方」を見る // ① クラスを定義 class MyGreeting implements Greeting { public void hello() { System.out.println("やあ"); } } // ② 使う Greeting g = new MyGreeting(); // 1回しか使わないのに2手間… // これを1つにまとめたい！
05

匿名クラスの書き方

HOW TO WRITE

書き方はこうです。new インターフェース名() { ここに中身 }。「new した直後に { } で中身を書く」のが匿名クラスの目印。クラス定義と new が合体しています。

new Type() { ... } 定義とnewを合体させる
// 前セクションと比べる
普通のやり方
クラスを別途定義してから new。2手間で行数も増える。
匿名クラス
new Greeting() { ... } でその場で完結。スッキリ。
// 匿名クラスで一気に書く Greeting g = new Greeting() { @Override public void hello() { System.out.println("やあ"); } }; // ← 最後のセミコロンを忘れずに！ g.hello(); // → やあ // 名前なしクラスを定義しつつ即 new した
POINT
匿名クラスの定番のハマりどころ：閉じ } の後にセミコロン ; が必要です。これは「new した式の文」だから。忘れるとエラーになるので注意。
06

現場での使われ方

REAL WORLD

匿名クラスが本当によく登場するのが「ボタンを押したら○○する」のような処理です。「この一回だけの動作」を、その場でサッと渡す。これが匿名クラスの黄金パターンです。

event handler ボタンの動作をその場で渡す
// よくある場面

「ボタンがクリックされた時の処理」を渡したい。でもその処理はこのボタン専用で一回きり。名前付きクラスを作るほどではない。

こういう時こそ匿名クラス。処理をその場に書いて即渡すのが自然です。

// ボタンクリック処理を匿名クラスで button.setOnClick(new ClickListener() { @Override public void onClick() { System.out.println("押されました！"); } }); // 「このボタン専用の動作」を // その場で書いて渡せる。これが定番。
TIP
実は最近のJavaでは、匿名クラスをさらに短く書ける「ラムダ式」という記法もあります（別レッスンで扱います）。でもその土台になっているのが匿名クラス。ここを理解しておくと、ラムダ式もスッと入ってきます。
