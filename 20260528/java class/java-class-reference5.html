NESTED CLASSES — intro & inner class / chapter 7-8

塊③に突入です。ネスト（nest）とは「入れ子」のこと。つまりクラスの中に、さらにクラスを書くのがネストされたクラスです。「そんなことできるの？」——できます。種類が4つあるので、この巻ではまず全体像を掴み、最初の1つ「内部クラス（Inner Class）」をじっくり学びます。あせらず一歩ずつ。

01

入れ子（ネスト）とは

WHAT IS NESTING

これまでクラスは「ファイルにひとつ」が基本でした。でも実はクラスの { } の中に、別のクラスを書けるのです。外側のクラスを外部クラス（Outer）、中のクラスをネストされたクラスと呼びます。

class in class クラスの中のクラス
// 入れ子のイメージ
Outer クラス（外側）
Inner クラス（中）

マトリョーシカや、引き出しの中の小箱を想像してください。外があって、その中に入っている関係です。

// クラスの中にクラス class Outer { // 外側のクラス class Inner { // 中のクラス（ネスト） // ここにもフィールドや // メソッドが普通に書ける } }
02

ネストされたクラスは4種類

FOUR TYPES

「クラスの中のクラス」には、書く場所と書き方によって4つの種類があります。今この地図を頭に入れておくと、Vol.5〜7で迷子になりません。この巻では①の内部クラスを扱います。

nested.map 4種類とそれぞれの担当巻

POINT
4つの違いは「どこに書くか」と「static が付くか」だけ。今は全部覚えなくて大丈夫。「4種類あるんだな」と知っていれば十分です。
03

内部クラス（Inner Class）

INNER CLASS

外部クラスの中に、static を付けずに書いたクラスが内部クラスです。一番の特徴は「外側にべったり寄り添う」こと。外部クラスのインスタンスとセットでしか存在できません。

inner class 外側に寄り添うクラス
// たとえ話：車とエンジン

車（Outer）の中のエンジン（Inner）を想像してください。エンジンは必ず特定の車に積まれた状態で存在します。エンジン単体がポツンと道に転がっていることはありません。

内部クラスも同じで、外部クラスのインスタンスがあって初めて作れるのです。

// 内部クラスの定義 class Car { // 外側＝車 String name = "愛車"; class Engine { // 内部＝エンジン void start() { System.out.println("始動！"); } } } // Engine は Car とセットで存在する
04

外側のフィールドを直接触れる

ACCESS OUTER

内部クラスのうれしい特権がこれ。内部クラスからは、外部クラスのフィールドやメソッドに自由にアクセスできます。private なものですら触れます。「べったり寄り添う」関係だからこそ許される特権です。

access privilege 外側の中身を自由に使える
// 動かすと…

内部クラス Engine から、外側 Car の name をそのまま読めます。引数で渡す必要すらありません。

出力
愛車のエンジン始動！

外側の name（"愛車"）を、内部クラスがそのまま使えています。

// 外側のフィールドを直接使う class Car { private String name = "愛車"; class Engine { void start() { // 外側の name をそのまま参照！ System.out.println(name + "のエンジン始動！"); } } }
TIP
この「外側のメンバーに自由に触れる」性質が、内部クラスを使う一番の理由です。外側と密接に協力する小さな部品を作りたい時に向いています。
05

内部クラスの作り方（new の書き方）

HOW TO CREATE

ここがハマりどころ。内部クラスを new する書き方は、ちょっと独特です。まず外側を new してから、その外側を通して内部を newします。「車を用意してから、その車のエンジンを作る」順番です。

outer.new Inner() 独特な生成の書き方
// 生成の手順
  1. まず外部クラスを new する
  2. その変数に .new を付けて内部クラスを作る

car.new Engine() という見慣れない形になります。最初は戸惑いますが、「車があってのエンジン」と思えば納得です。

// 内部クラスを生成して使う public static void main(String[] args) { // ① まず外側を new Car car = new Car(); // ② 外側を通して内部を new Car.Engine engine = car.new Engine(); engine.start(); // → 愛車のエンジン始動！ }
NG（やりがちなミス）
new Car.Engine() のように、外側のインスタンス無しで作ろうとするとエラー。
OK（正しい形）
car.new Engine() と、外側インスタンス経由で作る。
06

いつ使うか

WHEN TO USE

結論です。内部クラスは「外側のクラスとセットで動く、外には公開しない小さな部品」を作りたい時に使います。外側の中身を自由に触れるので、密接に協力する役割にぴったりです。

summary 内部クラスの使いどころ
// こんな時に内部クラス
  • 外側クラス専用の補助的な部品
  • 外側のフィールドを頻繁に使う
  • 外部に単独で見せる必要がない

逆に「単独でも使いたい」なら、内部クラスではなく普通のクラスか、Vol.7の static ネステッドが向いています。

// 現場でよくある例：リストの中身を管理 class MyList { // 外には見せない、内部専用の「節（ノード）」 class Node { Object value; Node next; } // MyList の中だけで Node を使い回す } // Node は MyList とべったりで構わない
