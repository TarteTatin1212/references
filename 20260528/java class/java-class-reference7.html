01

スタティックネステッドクラスとは

WHAT IT IS

外側のクラスの中に static class と書いたものが、スタティックネステッドクラスです。内部クラスとの見た目の違いはstatic がひとつあるだけ。でも意味は大きく変わります。外側のインスタンスがなくても作れるのです。

static class 外側から独立したネストクラス
// たとえ話：マンションと管理会社

マンション（Outer）の資料に「管理会社の情報」が載っています。でも管理会社はマンションとは別の独立した存在。資料に書かれているだけで、マンションがなくても会社は存在します。

staticネステッドも「外側の中に書いてあるけど、独立して動ける」クラスです。

// static を付けてネスト class Outer { static class Nested { // ← static 付き void hello() { System.out.println("独立してます"); } } } // Vol.5 の内部クラスとの差は static だけ
02

内部クラスとの決定的な違い

KEY DIFFERENCE

違いは大きく2つ。①外側のインスタンスが不要、②外側の（staticでない）フィールドには触れない。内部クラスが「外側べったり」だったのに対し、staticネステッドは「外側と距離を置く」関係です。

inner vs static 寄り添う / 距離を置く
// 関係の違い
内部クラス（Vol.5）
外側のインスタンスとセット。外側の private にも自由に触れる。べったり。
staticネステッド（今回）
外側インスタンス不要。外側のインスタンス変数には触れない。独立寄り。
// 触れる / 触れないの違い class Outer { int x = 10; // インスタンス変数 static int y = 20; // static 変数 static class Nested { void test() { // x は触れない（インスタンス変数だから） System.out.println(y); // ⭕ static の y はOK } } }
03

作り方はシンプル

HOW TO CREATE

内部クラスの car.new Engine() は独特でしたね。staticネステッドはもっと素直。外側のインスタンスがいらないので、new Outer.Nested() と普通に書けます。ここが内部クラスより楽なポイントです。

new Outer.Nested() 外側インスタンス不要で素直に作れる
// 作り方の比較
内部クラス（手間あり）
Outer o = new Outer();
o.new Inner();
staticネステッド（素直）
new Outer.Nested(); 一発でOK
// staticネステッドの生成 public static void main(String[] args) { // 外側を new しなくていい！ Outer.Nested n = new Outer.Nested(); n.hello(); // → 独立してます }
POINT
覚え方：static が付く＝外側から独立＝普通に new できる。staticのない内部クラスだけが「外側インスタンス経由」という特殊な作り方になる、と整理すると混乱しません。

05

内部クラスとの使い分け

WHICH TO USE

「内部クラスとstaticネステッド、どっちを使えば？」——結論はシンプル。外側のインスタンス変数を使いたいなら内部クラス、使わないならstaticネステッド。迷ったら static を付けるのが現場の鉄則です。

decision 迷ったら static
// 判断フロー

「外側のインスタンスの中身（static でないフィールド）を、このネストクラスから使いたい？」

YES → 内部クラス
外側にべったり寄り添う必要がある。
NO → staticネステッド
独立できるなら static のほうが無駄がない。
// 鉄則：迷ったら static を付ける // 理由： // static の方が外側との結びつきが弱く、 // メモリやバグの面で扱いやすい。 // 「外側の中身を本当に使う時だけ」 // static を外して内部クラスにする、 // という順序で考えると安全です。
TIP
現場では「とりあえず static ネステッドで書き始め、外側の中身が必要になったら static を外す」という進め方が多いです。最初から内部クラスにする必要はあまりありません。
