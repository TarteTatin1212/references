01

なぜ標準が便利なのか

WHY STANDARD

毎回 interface Calc {...} と自作するのは面倒です。そこでJavaは「よくあるパターンの関数型インターフェース」を最初から用意してくれています。これを使えば自分で定義しなくていい。ストリームもこれらを前提に作られています。

java.util.function標準で入っているパッケージ
// 4大インターフェースの役割
Function 入れて → 変換して出す
Consumer 受け取って → 消費する（戻り値なし）
Supplier 何も受け取らず → 供給する
Predicate 受け取って → true/falseで判定
// importするだけで使える
import java.util.function.Function;
import java.util.function.Consumer;
import java.util.function.Supplier;
import java.util.function.Predicate;

// 自作 interface は不要！
// この4つを覚えれば
// たいていの場面に対応できる
POINT —この4つはストリームのfilter・mapなどの中で実際に使われている主役です。ここで顔と名前を覚えておくと、Vol.7以降がぐっと楽になります。
02

Function（変換）

FUNCTION — TRANSFORM

Function は「入れたものを、別のものに変換して返す」係です。★★★ 最重要。apply メソッドを持ち、「Tを受け取ってRを返す」という形。たとえば「文字列を受け取って、その長さ（数値）を返す」のがFunctionの典型です。

Function<T, R>T入れて R返す
// 実行結果
5

"Hello"（文字列）を入れたら、長さの 5（数値）が返ってきます。
文字列 → 数値 へ「変換」しています。

// 文字列 → その長さ に変換
// String を受け取り Integer を返す
Function<String,Integer> len =
    s -> s.length();

System.out.println(
  len.apply("Hello")  // → 5
);
TIP —ストリームの .map() は、このFunctionを受け取ります。「mapは変換」＝「Functionを渡す」とセットで覚えてください。<T, R> は「T入れてR返す」の意味です。
03

Consumer（消費）

CONSUMER — USE IT

Consumer は「受け取って、何かするけど、何も返さない」係です。consume＝消費。典型は「受け取った値を画面に表示する」。表示したら終わり、返すものはない。accept メソッドを持ちます。

Consumer<T>T受け取る・戻り値なし
// 実行結果
こんにちは、田中さん

名前を受け取って、挨拶を表示。
表示したら役目は終わり。戻り値はありません。

// 受け取って表示するだけ
// String を受け取り 何も返さない
Consumer<String> greet =
    name -> System.out.println(
      "こんにちは、" + name + "さん"
    );

greet.accept("田中");
// → こんにちは、田中さん
TIP —ストリームの .forEach() は、このConsumerを受け取ります。「forEachは1個ずつ消費」＝「Consumerを渡す」。Vol.1で出た System.out::println も、実はConsumerでした。
04

Supplier（供給）

SUPPLIER — PROVIDE

Supply は「何も受け取らないけど、何かを供給して返す」係です。supply＝供給。引数なしで、値を1つ作って返す。典型は「現在時刻を返す」「ランダムな値を返す」など。get メソッドを持ちます。

Supplier<T>引数なし・T返す
// 実行結果（例）
ようこそ！

何も渡していないのに、文字列が出てきます。
「値を供給してくれる箱」のイメージです。

// 引数なしで値を供給
// 何も受け取らず String を返す
Supplier<String> msg =
    () -> "ようこそ！";

System.out.println(
  msg.get()  // → ようこそ！
);

// () は「引数なし」の印
POINT —引数がないので、ラムダは () -> ... と空のカッコから始まるのが特徴です。「何ももらわず、何かをくれる」——それがSupplier。Vol.10のOptionalでまた登場します。
05

Predicate（判定）

PREDICATE — TEST

Predicate は「受け取って、true か false を返す」判定係です。★★★ 最重要。predicate＝断定・判定。「この数は偶数か？」「この文字列は空か？」のような条件判定に使います。test メソッドを持ちます。

Predicate<T>T受け取る・true/false返す
// 実行結果
true
false

4は偶数なので true、7は偶数でないので false。
「Yes/No」を返すのがPredicateです。

// 偶数かどうかを判定
// Integer を受け取り boolean を返す
Predicate<Integer> isEven =
    n -> n % 2 == 0;

System.out.println(
  isEven.test(4)   // → true
);
System.out.println(
  isEven.test(7)   // → false
);
TIP —ストリームの .filter() は、このPredicateを受け取ります。「filterは条件で絞る」＝「Predicateを渡す」。Vol.1で見た n -> n % 2 == 0 は、まさにPredicateだったわけです。
