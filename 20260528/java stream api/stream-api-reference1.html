01

ストリームAPIとは

WHAT IS STREAM API

ストリームAPIは、リストや配列のデータを「流れ作業」で処理する仕組みです。Java 8から登場しました。「集めて、選んで、加工して、まとめる」——この一連の作業を、短く読みやすく書けるようになります。

stream = データの流れ 川に例えると分かりやすい
// イメージ図
SOURCE 源流リスト・配列
▼
中間操作選ぶ・変える
▼
終端操作集める・数える
// 言葉にするとこうです
// データを川に「流して」
// 途中で 選別・加工 して
// 最後に 結果を受け取る

list.stream()        // 流し始め
    .filter(...)     // 選ぶ
    .map(...)        // 変える
    .collect(...);   // まとめる
POINT — ストリームは「データそのもの」ではなく「データの流れ」です。元のリストは変更されません。流れの中でコピーを加工していくイメージです。
02

ビフォーアフター

BEFORE / AFTER

なぜストリームを学ぶのか。それは同じ処理がぐっと短く・読みやすくなるからです。「偶数だけ取り出して2倍にする」という処理を、従来の書き方とストリームで比べてみます。

同じ結果・違う書き方 forループ vs ストリーム
// BEFORE（従来のforループ）
入れ物（result）を用意する
forで1個ずつ取り出す
ifで偶数か判定
2倍してaddする

手順が多く、何をしたいか読み取りにくい。

// AFTER（ストリーム）
// 従来の書き方
List<Integer> result =
    new ArrayList<>();
for (int n : nums) {
  if (n % 2 == 0)
    result.add(n * 2);
}

// ストリームなら3行
List<Integer> r = nums.stream()
    .filter(n -> n % 2 == 0)
    .map(n -> n * 2)
    .toList();
POINT — ストリームは「やりたいこと」がそのまま並ぶのが魅力です。「選んで（filter）、変えて（map）、リストにする（toList）」と読めます。これが読みやすさの正体です。

04

3段階の流れ

THREE STAGES

ストリーム処理は必ず3段階で組み立てます。「生成 → 中間操作 → 終端操作」。この型を覚えれば、どんなストリームも同じ形に見えてきます。

生成 → 中間 → 終端 ストリームの黄金パターン
// 3段階の役割
① 生成 CREATE.stream()
▼
② 中間 INTERMEDIATE.filter() .map()
▼
③ 終端 TERMINAL.toList() .count()

②は何個でも繋げてOK。③は最後に1回だけ。

// 色分けして見ると
names.stream()              ①生成
     .filter(s -> ...)     ②中間
     .map(s -> ...)        ②中間
     .sorted()           ②中間
     .forEach(...);       ③終端

// ② は「途中の加工」
//    何回でも繋げられる
// ③ は「最後の出口」
//    ここで流れが終わる
POINT — 終端操作を書かないと、ストリームは1ミリも動きません。これは初学者が必ずハマる詰まりポイント。中間操作は「予約」、終端操作で「実行」されると覚えてください。

最初の1行を動かす

YOUR FIRST STREAM

理屈はここまで。さっそく動く最小のストリームを1つ書いてみましょう。リストの中身を1つずつ表示するだけの、いちばん簡単な例です。

Hello, Stream コピペで動く最小例
// 実行結果（コンソール）
りんご
みかん
ぶどう

リストの中身が、上から順に1行ずつ表示されます。

// Main.java
import java.util.List;

public class Main {
  public static void main(String[] a) {
    List<String> fruits =
      List.of("りんご",
              "みかん",
              "ぶどう");

    fruits.stream()
          .forEach(System.out::println);
  }
}
POINT — System.out::println という見慣れない書き方が出てきました。これがメソッド参照（Vol.5-6で解説）。今は「これも動く魔法」くらいで大丈夫。まず動かす体験が大事です。
