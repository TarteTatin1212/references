クラス編

OPTIONAL CLASS

プログラマを長年悩ませてきたnull地獄。「値が無いかもしれない」状況を安全に扱うために生まれたのがOptionalです。中身が「あるかもしれないし、無いかもしれない箱」。ストリームの findFirst など「結果が1個あるか無いか分からない」場面で必ず出会います。この巻でnull恐怖症を卒業しましょう。

01

nullの問題

THE NULL PROBLEM

まず敵を知りましょう。null は「何も無い」を表す特別な値。便利なようで、これに対してメソッドを呼ぶとNullPointerExceptionという有名なエラーで落ちます。初学者が最初にぶつかる壁です。

NullPointerException通称ヌルポ、永遠の敵
// なぜ落ちるか
値が null（何も無い）
null に .length() を呼ぶ
「無いものは操作できない」
→ NullPointerException
// よくあるクラッシュ
String name = null;

// null に対してメソッドを呼ぶと…
System.out.println(
  name.length()  // 💥 ここで落ちる
);

// NullPointerException
// 「nullのメソッドは呼べない」
POINT —NullPointerException（通称ヌルポ）は、Javaでもっとも遭遇するエラーのひとつ。原因は「nullかもしれない値を、チェックせずに使った」こと。Optionalはこのうっかりを防ぐ仕組みです。
02

Optionalとは

WHAT IS OPTIONAL

Optional は「値が入っているかもしれない箱」です。中身がある状態（present）と、空っぽの状態（empty）の2つ。「箱ごと受け取って、開ける前に中身があるか確認できる」のがミソ。いきなりnullで落ちることを防ぎます。

Optional<T>中身ありか空かの箱
// 2つの状態
📦 中身あり："りんご"
📦 空っぽ：empty

どちらの状態かを安全に確認してから開けられます。

// Optionalを作ってみる
// 中身ありの箱
Optional<String> a =
    Optional.of("りんご");

// 空っぽの箱
Optional<String> b =
    Optional.empty();

// 中身があるか確認
System.out.println(
  a.isPresent());  // true
POINT —Optionalの発想は「nullを直接扱わず、箱に包んで安全に運ぶ」こと。箱を開ける前に「中身ある？」と聞ける。この一手間が、ヌルポを劇的に減らします。これが神機能と呼ばれる理由です。
03

ストリームで出会う

MEET IN STREAMS

Optionalはストリームの終端操作の戻り値としてよく登場します。代表が findFirst（最初の1件を取る）。「条件に合う要素が1個も無いかもしれない」ので、結果はOptionalで返ってくるのです。

.findFirst()結果はOptionalで返る
// なぜOptionalなのか
条件に合う最初の1件を探す
でも1件も無いかもしれない
だから「箱」で返す
→ Optional<String>
// 最初の長い単語を探す
Optional<String> found =
    List.of("fig","apple")
        .stream()
        .filter(s -> s.length() > 3)
        .findFirst();

// "apple" が入った箱が返る
// もし条件に合う物が無ければ
// 空っぽの箱が返る
POINT —「なぜそのまま文字列を返さないの？」——それは「見つからなかった場合」があるから。nullを返すと受け取った側がうっかり落とす。Optionalなら「無いかもよ」と型で警告してくれる。親切設計です。
04

orElse（無ければ代替値）

OR ELSE

★★★ 最重要。orElse は「中身があればそれを、無ければ代わりの値を返す」メソッドです。これがOptionalの主役級の使い方。「見つかればその値、見つからなければデフォルト値」が1行で書けます。

.orElse(代替値)空のときの保険
// 実行結果
apple
該当なし

1つ目は見つかったので "apple"。
2つ目は空なので代替値 "該当なし"。

// 無ければデフォルト値を使う
// 見つかるケース
String r1 = List.of("apple")
    .stream().findFirst()
    .orElse("該当なし");
// → apple

// 見つからないケース
String r2 = List.<String>of()
    .stream().findFirst()
    .orElse("該当なし");
// → 該当なし（空だったので代替）
POINT —orElse を使えば「もし無ければこの値」を安全に書けます。nullチェックのif文を書く必要なし。findFirst().orElse("デフォルト") は現場で頻出の鉄板パターン。これだけ覚えれば初心者卒業です。
05

ifPresent（あれば実行）

IF PRESENT

ifPresent は「中身があるときだけ、処理を実行する」メソッドです。渡すのはConsumer（消費）。「値があれば表示、無ければ何もしない」が安全に書けます。空のときに何もしないので、ヌルポが起きません。

.ifPresent(Consumer)あるときだけ動く
// 実行結果
見つかった: apple

中身があったので表示。
空だったら、この行は何も起きません。

// 中身があるときだけ表示
List.of("apple","fig")
    .stream()
    .findFirst()
    .ifPresent(s ->
       System.out.println(
         "見つかった: " + s));

// → 見つかった: apple

// もし空なら何も表示されない
// （エラーにもならない）
TIP —使い分けの目安：値を「取り出して使う」なら orElse、値が「あるときだけ何かする」なら ifPresent。どちらも「箱を安全に開ける」ためのメソッド。この2つが使えれば、Optionalの実用は十分カバーできます。
06

やってはいけないこと

ANTI-PATTERNS

最後に注意点。Optionalを使う意味を消してしまうNG行為があります。代表が get() をいきなり呼ぶこと。これは「箱を確認せず開ける」のと同じで、空だと結局エラーで落ちます。本末転倒です。

get()の直呼びは危険★★ ありがちな罠
// NGとOK
NG：いきなりget()opt.get() 空だと例外で落ちる。Optionalの意味がない
OK：orElseやifPresentopt.orElse(代替) 空でも安全に処理できる
// 安全な書き方を選ぶ
// ✕ 危険：箱を確認せず開ける
String s = opt.get();
// 空だと NoSuchElementException

// ◯ 安全：代替値を用意
String s = opt.orElse("なし");

// ◯ 安全：あるときだけ処理
opt.ifPresent(
  v -> System.out.println(v));
POINT —鉄則：get() は基本使わない。代わりに orElse か ifPresent を使う。せっかく安全な箱（Optional）を受け取ったのに、get()でいきなり開けたら台無し。「箱は安全に開ける」——これがOptionalのお約束です。
