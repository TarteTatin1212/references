中間操作編

INTERMEDIATE OPERATIONS

ストリームの心臓部・前半です。中間操作とは、流れの途中でデータを加工する操作のこと。選ぶ（filter）・変える（map）・並べる（sorted）・重複を消す（distinct）——よく使う中間操作を一気に学びます。どれも「Stream を返す」ので、いくつでも . で繋げられます。ここが使えると、ストリームが一気に実用的になります。

CONTENTS
  • 01 filter（選ぶ）
  • 02 map（変える）
  • 03 sorted（並べる）
  • 04 distinct（重複削除）
  • 05 limit / skip（個数）

01

filter（選ぶ）

FILTER

★★★ 最重要。filter は「条件に合うものだけ残す」操作です。渡すのはPredicate（判定）。true になったものだけが、次へ流れていきます。

.filter(Predicate)条件で絞り込む
// 偶数だけ残す
入力1234
判定偶数か？
出力1234

偶数の 2 と 4 だけが残ります。

// 偶数だけ抽出
List.of(1,2,3,4)
    .stream()
    .filter(n -> n % 2 == 0)
    .forEach(System.out::println);

// → 2
// → 4

// true になった要素だけ通過
POINT —filterは「ふるい」です。条件（Predicate）を通った要素だけが下に落ちる。n % 2 == 0（偶数）、s.length() > 3（4文字以上）、!s.isEmpty()（空でない）——条件は自由自在です。
02

map（変える）

MAP

★★★ 最重要。map は「1つ1つを別のものに変換する」操作です。渡すのはFunction（変換）。要素の個数は変わらず、中身だけが変わります。

.map(Function)全要素を変換
// 全部を大文字に
入力abc
変換大文字に
出力ABC

個数は3個のまま。中身だけ変わります。

// 小文字 → 大文字に変換
List.of("a","b","c")
    .stream()
    .map(String::toUpperCase)
    .forEach(System.out::println);

// → A
// → B
// → C
POINT —filterとmapの違いを押さえましょう。filterは「数を減らす（中身はそのまま）」、mapは「中身を変える（数はそのまま）」。この2つがストリームの二大エース。String::toUpperCase のようにメソッド参照と相性抜群です。
03

sorted（並べる）

SORTED

sorted は「並び替える」操作です。引数なしで呼べば自然順（数値なら小さい順、文字なら辞書順）に並びます。逆順にしたいときは少し書き足します。

.sorted()並び替え
// 小さい順に並べる
入力312
並替昇順
出力123
// 昇順・降順
// 昇順（小さい順）
List.of(3,1,2).stream()
    .sorted()
    .forEach(System.out::println);
// → 1 2 3

// 降順（大きい順）
List.of(3,1,2).stream()
    .sorted(Comparator
            .reverseOrder())
    .forEach(System.out::println);
// → 3 2 1
TIP —降順は Comparator.reverseOrder() を渡します。最初は「昇順は sorted()、降順は sorted(Comparator.reverseOrder())」と丸暗記でOK。Comparatorは奥が深いので、今は2パターンだけ押さえれば十分です。
04

distinct（重複削除）

DISTINCT

distinct は「重複を取り除く」操作です。引数は不要。同じ値が複数あったら、1個だけ残してあとは消します。「ユニークな値だけ欲しい」ときの定番です。

.distinct()重複を1個にまとめる
// ダブりを消す
入力12233
処理重複削除
出力123
// 重複を除く
List.of(1,2,2,3,3)
    .stream()
    .distinct()
    .forEach(System.out::println);

// → 1
// → 2
// → 3

// ダブった 2 と 3 が1個ずつに
POINT —distinctは「同じかどうか」を equals で判定します。数値や文字列はそのまま使えて便利。「リストからダブりを消したい」は実務でも頻出。.distinct() 一発で済むのは超便利です。
05

limit / skip（個数で操作）

LIMIT / SKIP

limit は「先頭からN個だけ取る」、skip は「先頭からN個飛ばす」操作です。「上位3件だけ」「最初の1件を除く」のような場面で活躍します。ページング処理の基礎にもなります。

.limit(n) / .skip(n)個数で切り出す
// limit(2) と skip(2)
入力1234
limit21234
skip21234
// 先頭2個 / 先頭2個飛ばす
// 先頭から2個だけ
List.of(1,2,3,4).stream()
    .limit(2)
    .forEach(System.out::println);
// → 1 2

// 先頭2個を飛ばす
List.of(1,2,3,4).stream()
    .skip(2)
    .forEach(System.out::println);
// → 3 4
TIP —sorted() と limit() を組み合わせると「上位N件」が作れます。例：.sorted(reverseOrder()).limit(3) で「大きい順トップ3」。これは現場で超頻出の定番パターンです。
06

繋げて使う

CHAINING

中間操作の真価は「いくつでも繋げられる」こと。filter→map→sorted…と並べると、複雑な処理も1本の流れで書けます。上から順に読めば処理が分かるのが気持ちいいところ。

filter → map → sorted★★ 中間操作の合わせ技
// 実行結果
APPLE
GRAPE

5文字以上を選び、大文字化し、辞書順に。
"apple"と"grape"が残り、大文字＆ソート。

// 3つの中間操作を連結
List.of("apple","fig",
        "grape","kiwi")
    .stream()
    .filter(s -> s.length() >= 5)
    .map(String::toUpperCase)
    .sorted()
    .forEach(System.out::println);

// fig,kiwi は4文字以下で脱落
// apple,grape が残り大文字化
POINT —順番には意味があります。「選んでから変換」と「変換してから選ぶ」では結果が変わることも。基本は「filterで早めに減らす→mapで変換」が効率的。読みやすさと効率、両方の意味で順番を意識してください。
