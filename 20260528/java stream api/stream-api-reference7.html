01

ストリームAPIの全体像

THE BIG PICTURE

あらためて。ストリームAPIは「リストや配列を流れ作業で処理する仕組み」でした。Vol.1の復習を兼ねて、全体像をもう一度。データを流して、途中で加工して、最後に受け取る。この3拍子です。

流れ作業のイメージVol.1の復習＋深掘り
// データの流れ
① 生成リストから流す
▼
② 中間操作filter / map …
▼
③ 終端操作collect / forEach …
// 全体の型（黄金パターン）
list.stream()        ① 生成
    .filter(...)     ② 中間
    .map(...)        ② 中間
    .collect(...);   ③ 終端

// ②は何個でも繋げられる
// ③は最後に1回だけ
POINT —ストリームの嬉しさは「メソッドを . で繋いで書ける」こと。これをメソッドチェーンと呼びます。上から下へ読めば処理の流れがそのまま分かる。これが読みやすさの核心です。
02

Streamインターフェース

STREAM INTERFACE

流れの実体は Stream という型（インターフェース）です。list.stream() を呼ぶと、Stream型のオブジェクトが返ってきます。このStreamに対して、filterやmapを呼んでいく、というわけです。

Stream<T>流れを表す型
// Streamとは
Stream<String>
文字列が流れるストリーム
Stream<Integer>
数値が流れるストリーム
中身の型は<> で表す
// stream()の戻り値はStream型
List<String> list =
    List.of("a", "b", "c");

// stream() を呼ぶと…
Stream<String> s = list.stream();

// この s に対して
// filter や map を呼んでいく
s.forEach(System.out::println);
TIP —普段は変数に入れず list.stream().filter()... と直接繋ぎます。でも「途中に Stream という型が存在する」と知っておくと、エラーメッセージに Stream<...> が出ても慌てません。正体を知るのは強みです。
03

ストリームの生成

CREATING STREAMS

流れの作り方（生成）はいくつかあります。いちばん使うのは リスト.stream()。ほかに配列や、値を直接並べる方法も。まずは代表的な3つを押さえれば十分です。

stream() / of() / Arrays.stream()生成の代表3パターン
// 生成の使い分け
list.stream() リストから（最頻出）
Stream.of(...) 値を直接並べて
Arrays.stream(arr) 配列から
// 3つの生成方法
// ① リストから（いちばん使う）
List.of("a","b").stream();

// ② 値を直接並べて
Stream.of("a", "b", "c");

// ③ 配列から
String[] arr = {"a", "b"};
Arrays.stream(arr);
POINT —迷ったらlist.stream() だけ覚えればOK。現場の9割はこれです。配列なら Arrays.stream()、その場で値を並べたいなら Stream.of()、と必要になったら思い出す程度で大丈夫です。

04

3段階を体感する

FEEL THE THREE STAGES

実際に動く例で3段階を体感します。「3より大きい数だけ残して、2倍して、表示」。生成→中間→中間→終端、と流れを目で追ってください。

生成 → filter → map → forEach★★★ 流れの基本形
// 実行結果
8
10

[1,2,4,5] のうち 3より大きい 4と5 が残り、
2倍されて 8 と 10 が表示されます。

// 3段階が全部入った例
List.of(1, 2, 4, 5)
    .stream()              ①生成
    .filter(n -> n > 3)   ②中間：選ぶ
    .map(n -> n * 2)     ②中間：変える
    .forEach(            ③終端：表示
       System.out::println);

// 1,2 は filter で消える
// 4→8, 5→10 になる
POINT —filterに渡す n -> n > 3 はPredicate（判定）、mapに渡す n -> n * 2 はFunction（変換）、forEachに渡す System.out::println はConsumer（消費）。第1部の知識が全部使われています。土台が活きる瞬間です。
05

中間操作は「予約」

LAZY EVALUATION

超重要な性質。中間操作だけでは、何も実行されません。filterやmapは「こう処理してね」という予約にすぎず、終端操作が呼ばれて初めて、まとめて実行されます。ここは初学者が必ずハマる詰まりポイントです。

終端がなければ動かない★★★ 最大のハマりどころ
// 予約と実行
中間操作 = 予約（まだ動かない）
終端操作 = 実行（ここで動く）
終端がないと1ミリも動かない
// よくある「動かない」ミス
// ✕ 終端操作がない
List.of(1,2,3).stream()
    .map(n -> n * 2);
// → 何も起きない！
//   mapは「予約」されただけ

// ◯ 終端操作を付ける
List.of(1,2,3).stream()
    .map(n -> n * 2)
    .forEach(System.out::println);
// → 2 4 6 が表示される
POINT —「ストリームを書いたのに動かない！」のほとんどが終端操作の書き忘れです。鉄則：ストリームは必ず終端操作で締める。中間操作だけで終わっていないか、まず疑ってください。
06

使い捨てのルール

ONE-TIME USE

もう1つ大事な性質。ストリームは1回使ったら、もう使えません。終端操作を呼んだストリームは「閉じた」状態になり、再利用するとエラーになります。「流れ終わった川には、もう水はない」イメージです。

1ストリーム1終端★★ 意外なハマりどころ
// 使い捨ての考え方
ストリームは1回限りの流れ
終端を呼ぶと閉じる
使い回したいなら毎回 stream()

元のリストは残るので、また stream() し直せばOK。

// 使い回しはエラー
Stream<Integer> s =
    List.of(1,2,3).stream();

s.forEach(...);  // 1回目OK
s.forEach(...);  // ✕ エラー！
// 「stream has already been
//  operated upon or closed」

// ◯ もう一度 stream() し直す
list.stream().forEach(...);
list.stream().forEach(...);
TIP —「already been operated upon」というエラーが出たら、ストリームを使い回している合図です。ストリームは変数に取っておかず、その場で list.stream()... と書き切るのが安全。これも定番のお約束です。
