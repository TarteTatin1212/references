01

前編の振り返り

RECAP

まず前編のおさらい。メソッド参照は○○::△△ の形で、ラムダの「メソッド1個呼ぶだけ」を短くする記法でした。4種類あり、前編では①staticを学びました。残り3つをこの巻で扱います。

4種類のおさらい残り3つに進む
// 進捗マップ
①static 前編で完了 ✓
②型のインスタンス参照 ←この巻
③特定インスタンス参照 ←この巻
④コンストラクタ参照 ←この巻
// 共通の形は変わらない
// どれも :: を使う

② String::toUpperCase
③ System.out::println
④ ArrayList::new

// ::の「左」に注目すると
// 種類が見分けられる
POINT —覚えておくべき視点は1つだけ。「:: の左に何が来ているか」。型名なのか、特定の変数なのか、newなのか。ここを見れば種類が分かります。左を見る——これがこの巻の合言葉です。
02

型のインスタンスメソッド参照

TYPE INSTANCE REF

2つ目、型のインスタンスメソッド参照です。型名::メソッド名。前編のstaticと見た目は同じですが、参照するのがインスタンスメソッド（オブジェクトに対して呼ぶメソッド）という点が違います。「受け取った値そのものに対してメソッドを呼ぶ」型です。

String::toUpperCase受け取った値に対して呼ぶ
// 仕組み
s -> s.toUpperCase()
▼
String::toUpperCase

「受け取ったsに対して、sのメソッドを呼ぶ」
→ 型名だけ書けばよい。

// 受け取った値のメソッドを呼ぶ
// ラムダ式
Function<String,String> u1 =
    s -> s.toUpperCase();

// 型のインスタンスメソッド参照
Function<String,String> u2 =
    String::toUpperCase;

System.out.println(
  u2.apply("hello")); // HELLO
TIP —ポイントは「受け取る引数（s）が、メソッドを呼ぶ側になる」こと。s.toUpperCase() の s がそのまま主語です。だから String::toUpperCase で「受け取ったStringのtoUpperCaseを呼ぶ」と読めます。
03

特定インスタンスのメソッド参照

SPECIFIC INSTANCE REF

3つ目、特定インスタンスのメソッド参照です。変数名::メソッド名。今度は :: の左が「特定のオブジェクト（変数）」。Vol.1で出てきた System.out::println がまさにこれです。

System.out::printlnあの謎の記法の正体
// 実行結果
りんご
みかん

System.out という特定のオブジェクトの
printlnを参照しています。

// 特定オブジェクトのメソッド参照
// ラムダ式
Consumer<String> c1 =
    s -> System.out.println(s);

// 特定インスタンスのメソッド参照
Consumer<String> c2 =
    System.out::println;

List.of("りんご","みかん")
    .forEach(System.out::println);
POINT —System.out は「標準出力」という具体的なオブジェクト。それの println を呼ぶので System.out::println。Vol.1で「魔法」と言ったこの記法、もう正体が分かりましたね。伏線回収完了です。
04

2つの違いを見分ける

SPOT THE DIFFERENCE

②と③、どちらも見た目は ○○::メソッド。違いは「::の左が、型名か、変数か」だけです。ここが最大の詰まりポイントなので、表でスッキリ整理します。

型名 vs 変数★★ ここが混乱の元
// 左を見れば分かる
String::toUpperCase
左が「型名」→ 受け取った値が主語
System.out::println
左が「変数」→ その変数が主語
種類::の左主語は
②型String受け取った引数
③特定System.outその変数自身

// 大文字始まりの型名か、
// 小文字の変数か、で見分ける

TIP —ざっくりした見分け方：「左が大文字始まりの型名 → ②」「左が具体的な変数 → ③」。最初は混乱して当然です。書いているうちに体が覚えます。分からなければラムダで書けばいい、という逃げ道も忘れずに。
05

コンストラクタ参照

CONSTRUCTOR REF

最後、4つ目のコンストラクタ参照です。クラス名::new。:: の右が new なのが特徴。「新しいインスタンスを作る」処理を参照します。::new を見たらコンストラクタ参照、と覚えれば一発です。

ArrayList::new:: の右が new
// 仕組み
() -> new ArrayList<>()
▼
ArrayList::new

「newするだけ」のラムダを短縮。

// インスタンス生成を参照
// ラムダ式
Supplier<List<String>> s1 =
    () -> new ArrayList<>();

// コンストラクタ参照（同じ意味）
Supplier<List<String>> s2 =
    ArrayList::new;

List<String> list = s2.get();
// 新しい空のリストができる
POINT —引数なしで new するなら前巻のSupplierと相性抜群。ArrayList::new は「呼ばれたら新しいArrayListを供給する」。ストリームのcollectで器を用意するときなどに登場します（Vol.11で再会）。

4種まとめ

ALL FOUR

土台パート最後の総まとめ。メソッド参照4種を一覧にします。完璧に暗記する必要はありません。「こういう4つがあって、見た目はどれも ::」と分かっていれば十分です。

メソッド参照 全4種★★ 保存版

左が変数? → ③特定インスタンス
