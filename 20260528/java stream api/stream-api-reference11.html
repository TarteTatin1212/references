01

Collectorsとは

WHAT IS COLLECTORS

Vol.9で collect をちらっと見ました。その collect に渡す「集め方のレシピ」が Collectors です。Collectors.toList() でリスト化、groupingBy でグループ分け…と、「集めた結果をどんな形にするか」を指定します。

collect(Collectors.○○)集め方のレシピ集
// Collectorsでできること
toList リストにまとめる
groupingBy 条件でグループ分け
toMap キーと値のマップに
joining 文字列を連結
// collectに「レシピ」を渡す
import static java.util.stream.
       Collectors.*;

list.stream()
    .collect(toList());

// collect の中に
// Collectorsのメソッドを入れる
// = 「こう集めてね」の指示
POINT —collect は「集める」終端操作、Collectors は「どう集めるかのレシピ」。この2つはセット。collect(Collectors.groupingBy(...)) のように組み合わせます。ストリームの集大成とも言える機能です。
02

groupingBy（グループ分け）

GROUPING BY

★★★ 実務最頻出。groupingBy は「ある基準でグループ分けする」レシピです。「文字数ごとに分ける」「カテゴリごとに分ける」など。結果はMap（キー→グループのリスト）で返ります。

groupingBy(分類の基準)仲間分けの神機能
// 文字数でグループ分け
3 → [fig]
5 → [apple, grape]
4 → [kiwi]

「文字数」をキーに、同じ長さの単語が集まります。

// 文字数ごとにグループ化
Map<Integer, List<String>> g =
  List.of("fig","apple",
          "grape","kiwi")
    .stream()
    .collect(groupingBy(
        String::length));

System.out.println(g);
// {3=[fig], 4=[kiwi],
//  5=[apple, grape]}
POINT —渡すのは「何を基準に分けるか」のFunction。String::length なら文字数、商品::getカテゴリ ならカテゴリ別。「○○ごとに分類して集計」は現場で超頻出。ループで書くと長い処理が、1行で書けてしまいます。
03

toMap（マップ化）

TO MAP

toMap は「キーと値のペアのMapを作る」レシピです。「単語→その文字数」のような対応表を作れます。引数は2つ——キーの作り方と値の作り方を、それぞれFunctionで渡します。

toMap(キー, 値)対応表を作る
// 単語 → 文字数の表
"fig" → 3
"apple" → 5
"kiwi" → 4

単語をキー、文字数を値にしたMapができます。

// 単語と文字数の対応表
Map<String, Integer> m =
  List.of("fig","apple","kiwi")
    .stream()
    .collect(toMap(
        s -> s,           // キー：単語
        String::length     // 値：文字数
    ));

System.out.println(m);
// {fig=3, apple=5, kiwi=4}
TIP —toMap の注意点：キーが重複するとエラーになります。「同じキーが2回出る」データには使えません（その場合はgroupingByの出番）。s -> s は「受け取った値をそのままキーにする」意味。よく使う書き方です。
04

joining（文字列連結）

JOINING

joining は「文字列を1本に繋げる」レシピです。区切り文字も指定できます。「リストの要素をカンマ区切りで1つの文字列に」——CSVやメッセージ作成でよく使う、地味だけど便利な機能です。

joining(", ")区切り文字で連結
// 実行結果
りんご, みかん, ぶどう

3つの単語が、カンマ区切りで1本の文字列に。

// カンマ区切りで連結
String joined =
  List.of("りんご","みかん",
          "ぶどう")
    .stream()
    .collect(joining(", "));

System.out.println(joined);
// → りんご, みかん, ぶどう

// joining() だけなら区切りなし連結
POINT —これでCollectorsの主要4つ（toList・groupingBy・toMap・joining）が揃いました。入門段階では groupingBy と joining が書ければ十分。残りは「こういうのもある」程度で、必要になったらこの巻に戻ってくればOKです。次はいよいよQ&Aです。
05

困った時のQ&A

TROUBLESHOOTING Q&A
ここからはQ&A形式。シリーズ全体でつまずきやすい「あるある」を10個集めました。エラーで詰まったとき、思い出せないとき、ここを開けば解決の糸口が見つかるはずです。辞書として活用してください。
Q.01

ストリームを書いたのに、何も起きません

A. ANSWER

終端操作の書き忘れが9割です。filterやmapなどの中間操作は「予約」にすぎず、終端操作（forEach・toList・count等）を書いて初めて実行されます。

NG：中間操作だけlist.stream().map(n -> n * 2); ← 予約だけで動かない
OK：終端操作で締めるlist.stream().map(n -> n * 2).forEach(...);
TIP —「動かない」と思ったら、まず末尾に終端操作があるかを確認。これが鉄則です。
Q.02

「stream has already been operated upon or closed」と出ます

A. ANSWER

ストリームを使い回しているサインです。ストリームは1回限りの流れ。終端操作を呼ぶと閉じてしまい、二度は使えません。

// ✕ 変数に取って2回使う
Stream<Integer> s = list.stream();
s.forEach(...);  // 1回目
s.count();      // ✕ 2回目で例外

// ◯ その都度 stream() し直す
list.stream().forEach(...);
list.stream().count();
POINT —ストリームは変数に保存せず、その場で書き切るのが安全。元のリストは何度でも stream() できます。
Q.03

ラムダ式で「returnが必要」と怒られます

A. ANSWER

波カッコ { } とreturnの関係が原因です。鉄則は「{ } を書いたらreturnも書く。{ } を書かないならreturnも書かない」。この2つは必ずセットです。

NG：{ }なのにreturnなしn -> { n * 2; } ← 値が返らずエラー
OK：1行なら両方省略n -> n * 2
OK：複数行なら両方書くn -> { int r = n*2; return r; }
TIP —1行で済むなら波カッコもreturnも省くのがいちばんスッキリします。
Q.04

filterとmap、どっちを使えばいいか分かりません

A. ANSWER

役割が違います。filterは「数を減らす（中身はそのまま）」、mapは「中身を変える（数はそのまま）」。やりたいことで選びます。

filter
条件で絞り込みたい
例：偶数だけ残す
map
各要素を変換したい
例：全部2倍にする
POINT —「選ぶならfilter、変えるならmap」。両方使うなら filter().map() と繋げばOKです。
Q.05

メソッド参照がうまく書けません

A. ANSWER

無理に使う必要はありません。メソッド参照はラムダの「短縮形」。書けなければラムダのままで全く問題ないです。使えるのは「メソッドを1個呼ぶだけ」のときに限られます。

OK：メソッド1個だけs -> s.toUpperCase() → String::toUpperCase
参照にできない：計算が混じるn -> n * 2 + 1 ← ラムダのままが正解
TIP —IDEが「メソッド参照にできます」と提案してくれることも。提案に乗るのがラクで確実です。
Q.06

NullPointerExceptionが出て落ちます

A. ANSWER

null（何も無い値）に対してメソッドを呼んだサインです。特にストリームの findFirst などの結果はOptionalで返るので、orElse や ifPresent で安全に開きましょう。

NG：いきなりget()opt.get() ← 空だと例外
OK：代替値を用意opt.orElse("デフォルト")
POINT —Optionalの get() は基本使わない。orElseかifPresentが安全策です（Vol.10参照）。
Q.07

toMapで「Duplicate key」エラーが出ます

A. ANSWER

toMapはキーが重複すると落ちます。同じキーが2回以上出るデータには使えません。「グループ分けしたい」のが本来の目的なら、groupingByを使うのが正解です。

NG：キーが重複しうるのにtoMap文字数をキーにすると、同じ長さの単語で衝突
OK：同じキーをまとめたいならgroupingBygroupingBy(String::length)
TIP —1対1の対応表ならtoMap、1対多のグループ分けならgroupingBy。目的で選びましょう。
Q.08

合計を出したいのですが、reduceが難しいです

A. ANSWER

単純な合計ならreduceより簡単な方法があります。数値に変換してから sum() を使うと読みやすいです。

// 難しめ：reduce
.reduce(0, (a,b) -> a + b);

// 簡単：mapToInt + sum
list.stream()
    .mapToInt(n -> n)
    .sum();  // → 合計
POINT —合計・平均などは mapToInt で数値ストリームにして sum()・average() が分かりやすいです。
Q.09

「上位3件」を取りたいです

A. ANSWER

sorted（並び替え）とlimit（個数制限）の合わせ技が定番です。「大きい順に並べて、先頭3つ」という流れになります。

list.stream()
    .sorted(Comparator
            .reverseOrder())  // 大きい順
    .limit(3)                 // 先頭3つ
    .toList();
TIP —sorted → limitはランキング作成の鉄板。小さい順トップ3なら sorted() だけでOKです。
Q.10

結局、最低限どれを覚えればいいですか？

A. ANSWER

これだけ覚えれば初心者卒業です。欲張らず、まずはこの5つを使いこなしましょう。

① filter — 条件で絞る（Predicate）
② map — 変換する（Function）
③ forEach — 1つずつ処理（Consumer）
④ toList — 結果をリスト化
⑤ orElse — Optionalを安全に開く
POINT —この5つで実務の8割はカバーできます。残りは必要になったとき、この11巻を辞書として開けばいい。それで十分です。
