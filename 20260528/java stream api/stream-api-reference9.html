終端操作編

TERMINAL OPERATIONS

ストリームの心臓部・後半です。終端操作とは、流れを実行して結果を取り出す操作のこと。これを呼んで初めてストリームは動きます（Vol.7の復習）。1つずつ処理（forEach）・リストにまとめる（collect）・数える（count）・合計する（reduce）——出口の代表選手を学びます。これでストリームが「使える道具」になります。

01

forEach（1つずつ処理）

FOREACH

forEach は「流れてきた要素を1つずつ処理する」終端操作です。渡すのはConsumer（消費）。いちばん馴染みのある終端操作で、これまでの巻でも表示によく使ってきました。

.forEach(Consumer)1個ずつ消費する
// 実行結果
りんご
みかん

流れてきた要素を1つずつ表示。
戻り値はなし（voidの終端）。

// 1つずつ表示
List.of("りんご","みかん")
    .stream()
    .forEach(System.out::println);

// → りんご
// → みかん

// forEach は結果を返さない
// 「処理して終わり」の終端
POINT —forEachは「結果を返さず、副作用（表示・保存など）を起こす」終端です。逆に「加工した結果をリストで欲しい」ときはforEachではなく、次のcollectを使います。使い分けが大事です。
02

collect / toList

COLLECT

★★★ 最重要。collect（や toList）は「流れた結果をリストにまとめる」終端操作です。ストリームで加工した結果を、また使える形（List）に戻す。実務でいちばん使う終端と言っても過言ではありません。

.toList() / .collect(...)結果をリスト化
// 実行結果
[2, 4, 6]

2倍した結果が、新しいListになって返ります。
変数に入れて、後で使えます。

// 結果をListとして受け取る
List<Integer> result =
    List.of(1,2,3).stream()
        .map(n -> n * 2)
        .toList();

System.out.println(result);
// → [2, 4, 6]

// 昔ながらの書き方なら
// .collect(Collectors.toList())
POINT —新しいJavaなら.toList() が簡単でおすすめ。古いコードでは .collect(Collectors.toList()) という形も見ます（同じ意味）。collectはもっと色々できる強力な終端で、Vol.11のCollectorsで本領を発揮します。
03

count（数える）

COUNT

count は「要素の個数を数える」終端操作です。引数なしで、流れてきた要素の数を long で返します。filterと組み合わせると「条件に合うものが何個あるか」が一発で分かります。

.count()個数を返す
// 実行結果
2

[1,2,3,4] のうち偶数は 2個（2と4）。
その個数 2 が返ります。

// 偶数が何個あるか数える
long cnt = List.of(1,2,3,4)
    .stream()
    .filter(n -> n % 2 == 0)
    .count();

System.out.println(cnt);
// → 2

// filter で絞って count で数える
// は定番の組み合わせ
TIP —filter(...).count() は「条件に合うものの数」を数える鉄板パターン。「在庫が10個未満の商品は何件？」のような集計が1行で書けます。戻り値が long 型（intより大きい整数）なのも覚えておくと安心です。
04

anyMatch / allMatch（判定）

MATCHING

anyMatch は「1つでも条件に合えばtrue」、allMatch は「全部合えばtrue」を返す終端操作です。渡すのはPredicate。「1件でも該当する？」「全部クリアしてる？」のチェックに使います。

.anyMatch() / .allMatch()true/falseで答える
// 違いを整理
anyMatch 1つでも合えば true
allMatch 全部合えば true
noneMatch 1つも合わなければ true
// 偶数が含まれるか / 全部偶数か
List<Integer> nums =
    List.of(2, 4, 5);

// 1つでも偶数ある？
nums.stream()
    .anyMatch(n -> n % 2 == 0);
// → true（2,4がある）

// 全部偶数？
nums.stream()
    .allMatch(n -> n % 2 == 0);
// → false（5が奇数）
POINT —3兄弟で覚えましょう。any（どれか）・all（すべて）・none（どれも〜ない）。「不良品が1つでもある？」は anyMatch、「全員合格？」は allMatch。ループを書かずに判定できるのがスマートです。
05

reduce（畳み込み）

REDUCE

reduce は「全要素を1つの値にまとめる」終端操作です。合計・最大・連結など。少し難しいので、まずは「合計を出す」典型例だけ押さえれば十分です。「隣同士を順番にくっつけていく」イメージ。

.reduce(初期値, (a,b)->...)★ ここは無理しなくてOK
// 合計の流れ
初期値 0 から始める
0+1=1 → 1+2=3 → 3+3=6
最後に残った 6 が答え
// 合計を求める
int sum = List.of(1,2,3)
    .stream()
    .reduce(0, (a, b) -> a + b);

System.out.println(sum);
// → 6

// 0 が初期値（スタート地点）
// (a,b)->a+b で順番に足す
TIP —正直、合計だけなら専用の簡単な方法があります。数値なら .mapToInt(n -> n).sum() の方が読みやすいことも。reduceは「合計以外の集約もできる万能選手」ですが、入門段階では「こういうものがある」程度でOK。無理は禁物です。

終端操作 早見表

CHEAT SHEET

終端操作を一覧で整理します。「何が欲しいか」で選ぶと覚えやすい。リストが欲しいならtoList、数が欲しいならcount、Yes/NoならanyMatch…という具合です。

終端操作まとめ★★ 保存版
// 欲しいもので選ぶ
リストが欲しい → toList
個数が欲しい → count
Yes/Noが欲しい → anyMatch等
1つずつ処理したい → forEach
操作戻り値用途
forEachなし1つずつ処理

countlong個数

POINT —もう一度、ストリームの鉄則。終端操作を書いて初めて流れが動く。中間操作（Vol.8）でいくら加工しても、終端（この巻）で締めないと何も起きません。「中間で加工、終端で取り出す」——この役割分担が腹落ちすれば、ストリームはもうあなたのものです。
