01

国際化とは何か

WHAT IS i18n

国際化とは、1つのプログラムを、複数の言語・地域にそのまま対応できる形にしておくことです。「日本語版」「英語版」と別々に作るのではなく、中身は1つ、表示だけ切り替える。これが基本の考え方です。

i18n 「internationalization」は i と n の間に18文字あるので i18n と略します
// 同じアプリ、相手によって表示が変わる
🇯🇵
日本語
ja_JP
ようこそ、田中さん
🇺🇸
英語（米国）
en_US
Welcome, Tanaka
🇫🇷
フランス語
fr_FR
Bienvenue, Tanaka
プログラム本体は1つだけ
// 言葉を直接書かない（これがコツ）
// ↓ こうはしない
System.out.println("ようこそ");

// ↓ こうする：「welcome というキー」を
//   その人の言語で取り出す
String msg = messages.getString("welcome");
System.out.println(msg);

// → 日本語の人には「ようこそ」
// → 英語の人には  「Welcome」
//   同じコードのまま、表示だけ変わる
POINT —
国際化のいちばんの肝は「言葉をコードに直接書かない」こと。文章は外部のファイルに追い出し、コードからはキー（合言葉）で呼び出します。この発想さえつかめば、あとは応用です。
02

なぜ国際化するのか

WHY i18n

「日本語だけで充分」と思うかもしれません。でも国際化は外国語対応だけの話ではありません。日付・数値・通貨の「書き方」も国によって違う。これを手作業で揃えると地獄を見ます。

国ごとの「当たり前」の違い 同じ「2024年3月5日」「1234567.89」でもこんなに違う
// 同じ意味なのに表記がバラバラ
国日付金額
日本2024/03/05¥1,234,568
米国3/5/2024$1,234,567.89
独国05.03.20241.234.567,89 €
仏国05/03/20241 234 567,89 €
ドイツでは小数点が「,」、桁区切りが「.」。日本と真逆です。
手作業でやろうとすると…
// こんなif文の山を書くハメに…（やりたくない）
if (country.equals("JP")) {
    format = "yyyy/MM/dd";
} else if (country.equals("US")) {
    format = "MM/dd/yyyy";
} else if (country.equals("DE")) {
    format = "dd.MM.yyyy";
}
// …国の数だけ増え続ける。破綻します

// Javaの国際化なら、これで済む ↓
// Localeを渡すだけ。あとはJavaがやってくれる

Javaには「その国の流儀を全部知っているライブラリ」が標準で入っています。国際化を学ぶとは、この賢いライブラリの呼び出し方を覚えることでもあります。自分でif文を書かない。それが現場の鉄則です。

03

i18nとl10n

i18n vs l10n

よく似た2つの言葉を整理します。国際化（i18n）は「箱を作る」、地域化（l10n）は「箱に中身を入れる」とイメージすると分かりやすいです。

i18n / l10n internationalization と localization
// 2つの役割分担
i18n（国際化）
  • 仕組み・土台づくり
  • 言葉を外に追い出す設計
  • プログラマの仕事
  • 一度やればOK
l10n（地域化）
  • 各言語の中身づくり
  • 実際の翻訳・調整
  • 翻訳者と協力
  • 言語を足すたび行う
順番のイメージ
① i18n：言葉を外部ファイルに追い出す土台を作る
↓
② l10n（日本語）：日本語の中身を用意する
↓
③ l10n（英語）：英語の中身を用意する
↓
必要なら④フランス語、⑤中国語…と足していく
POINT —
覚え方は簡単。i18n が先、l10n が後。土台（i18n）がしっかりしていれば、後から言語を足す（l10n）のは楽になります。このシリーズは主にi18nの作り方を扱います。
04

Localeという主役

THE Locale CLASS

国際化の世界ですべての中心にいるのがLocaleクラスです。これは「言語＋国」をひとまとめにした“宛先ラベル”のようなもの。「この人は日本語の日本の人」という情報を1つにまとめています。

java.util.Locale 言語コード（小文字）＋ 国コード（大文字）
// Localeの作り方いろいろ
Locale意味
Locale.JAPAN日本語・日本
Locale.US英語・米国
Locale.FRANCEフランス語・仏
Locale.JAPANESE日本語（国は不問）
Locale.ENGLISH英語（国は不問）
よく使う組み合わせは定数で用意されています。覚えなくても Locale. と打てば候補が出ます。
Locale.java
import java.util.Locale;

// ① 用意された定数を使う（いちばん簡単）
Locale jp = Locale.JAPAN;
Locale us = Locale.US;

// ② 自分で組み立てる（言語, 国）
Locale fr = Locale.of("fr", "FR");

// ③ 今このPCの設定を取得する
Locale now = Locale.getDefault();
System.out.println(now);
// → ja_JP （日本のPCならこう出る）

// 中身を確認
System.out.println(jp.getLanguage()); // ja
System.out.println(jp.getCountry());  // JP
TIP —
Locale.of("fr", "FR") は新しめの書き方です。古いコードでは new Locale("fr", "FR") も見かけますが、これから書くなら Locale.of(...) がおすすめ。意味は同じです。
05

最初の一歩

HELLO, i18n

理屈はここまで。実際に「Localeを変えると表示が変わる」のを体験しましょう。日付フォーマットを例にすると、たった数行で国ごとの違いが出せるのが分かります。

First Step Localeを差し替えるだけで結果が変わる
// 実行結果
渡したLocale出力
Locale.JAPAN2024年3月5日
Locale.USMar 5, 2024
Locale.FRANCE5 mars 2024
同じ日付データ。Localeを変えただけでこの違い。自分でif文は1つも書いていません。これが国際化の威力です。
HelloI18n.java
import java.time.LocalDate;
import java.time.format.DateTimeFormatter;
import java.time.format.FormatStyle;
import java.util.Locale;

public class HelloI18n {
  public static void main(String[] args) {
    LocalDate day = LocalDate.of(2024,3,5);

    show(day, Locale.JAPAN);
    show(day, Locale.US);
    show(day, Locale.FRANCE);
  }

  static void show(LocalDate d, Locale loc) {
    DateTimeFormatter f =
      DateTimeFormatter
        .ofLocalizedDate(FormatStyle.MEDIUM)
        .withLocale(loc);
    System.out.println(d.format(f));
  }
}

細かい文法はまだ分からなくて大丈夫です。「Localeを渡す → Javaが国の流儀で整えてくれる」という流れだけ感じ取れれば、この巻の目的は達成です。詳しい使い方はVol.4以降でじっくり扱います。
