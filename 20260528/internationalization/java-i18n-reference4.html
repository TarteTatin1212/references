01

日付を表すLocalDate

LocalDate BASICS

まず日付そのものを表すLocalDateと仲良くなりましょう。「2024年3月5日」という事実を持つ箱です。この時点では表示形式はまだ気にしません。「データ」と「見た目」を分けて考えるのがコツです。

java.time.LocalDate 年月日を持つ、シンプルな日付の箱
// 3兄弟を押さえる
クラス持つもの
LocalDate日付だけ（年月日）
LocalTime時刻だけ（時分秒）
LocalDateTime日付＋時刻
この巻は主に LocalDate を使います。仲間がいることだけ覚えておけば十分です。
LocalDate の作り方
import java.time.LocalDate;

// ① 今日
LocalDate today = LocalDate.now();

// ② 指定して作る（年, 月, 日）
LocalDate day = LocalDate.of(2024, 3, 5);

// ③ 中身を取り出す
System.out.println(day.getYear());        // 2024
System.out.println(day.getMonthValue());  // 3
System.out.println(day.getDayOfMonth()); // 5

// ④ 計算もできる（元は変わらない）
LocalDate next = day.plusDays(10); // 3/15
// day自身は3/5のまま。新しい日付が返る
POINT —
plusDays などの計算は元のLocalDateを変えず、新しい日付を返します（これを「不変＝イミュータブル」と呼びます）。day.plusDays(10) しても day 自身は3/5のまま。戻り値を受け取り忘れるのが定番ミスです。
02

DateTimeFormatter

THE FORMATTER

日付を「見た目の文字列」に変える係がDateTimeFormatterです。「データ（LocalDate）に整形係（Formatter）を渡して、文字列をもらう」。この役割分担が日付フォーマットの基本構造です。

java.time.format.DateTimeFormatter 日付 × 整形係 = 表示用の文字列
// 役割分担のイメージ
① LocalDate＝日付の「事実」を持つ
② DateTimeFormatter＝「どう見せるか」を決める
③ date.format(formatter) で文字列化
④ FormatterにLocaleを仕込むと国際化される
基本の組み立て
import java.time.LocalDate;
import java.time.format.DateTimeFormatter;

LocalDate day = LocalDate.of(2024, 3, 5);

// 整形係を用意
DateTimeFormatter f =
    DateTimeFormatter.ofPattern("yyyy/MM/dd");

// 日付に整形係を渡す → 文字列が返る
String text = day.format(f);
System.out.println(text); // 2024/03/05

// 「日付が整形係を使って自分を文字にする」
// という向きで読むと覚えやすい
TIP —
呼ぶ向きは日付.format(整形係)。「整形係.format(日付)」ではありません。主役は日付、整形係は道具。「日付が自分を文字に変える」とイメージすると順番を忘れません。
03

FormatStyleで簡単整形

LOCALIZED STYLES

国際化でいちばん楽なのがFormatStyleです。「短め・普通・長め」のスタイルとLocaleを指定するだけ。自分で「yyyy年MM月」などと書かなくても、その国の人が自然に読める形にJavaが整えてくれます。

ofLocalizedDate(FormatStyle).withLocale(...) 国ごとの「ちょうどいい書き方」を自動選択
// MEDIUM スタイルでの各国表示
JAPAN→2024年3月5日
US→Mar 5, 2024
FRANCE→5 mars 2024
GERMANY→05.03.2024
スタイルは SHORT / MEDIUM / LONG / FULL の4段階。長いほど丁寧な表記になります。
StyleExample.java
import java.time.*;
import java.time.format.*;
import java.util.Locale;

LocalDate d = LocalDate.of(2024,3,5);

// スタイル + Locale で整形係を作る
DateTimeFormatter f =
    DateTimeFormatter
      .ofLocalizedDate(FormatStyle.MEDIUM)
      .withLocale(Locale.JAPAN);

System.out.println(d.format(f));
// → 2024年3月5日

// withLocale を US に変えれば Mar 5, 2024
// FRANCE なら 5 mars 2024
// → 自分で書式を書かなくていい！
POINT —
国際化したい日付表示は、まず FormatStyle を第一候補にしましょう。「その国の人が読み慣れた形」をJavaが知っているので、自分で yyyy/MM/dd と決め打ちするより安全で自然。決め打ちは国際化の敵です。
04

自分でパターン指定

CUSTOM PATTERN

「どうしてもこの形にしたい」時はofPatternで書式を自分で指定します。記号の意味さえ分かれば自由自在。ただし国際化の観点では、固定書式は「全世界で同じ見た目になる」点に注意が必要です。

ofPattern("...") y=年 M=月 d=日、大文字小文字に意味がある
// よく使うパターン記号
記号意味例
yyyy年（4桁）2024
MM月（2桁）03
dd日（2桁）05
HH時（24h）14
mm分30
E曜日火
PatternExample.java
LocalDate d = LocalDate.of(2024,3,5);

// 固定の書式を指定
DateTimeFormatter f =
    DateTimeFormatter.ofPattern("yyyy/MM/dd");
System.out.println(d.format(f)); // 2024/03/05

// 曜日を含めて、Localeで曜日名を訳す
DateTimeFormatter jp =
    DateTimeFormatter
      .ofPattern("yyyy年M月d日(E)", Locale.JAPAN);
System.out.println(d.format(jp));
// → 2024年3月5日(火)

// ofPattern にも Locale を渡せる。
// すると「火」「Tue」「mar.」と曜日が訳される
TIP —
記号は大文字小文字で意味が変わります。MMは月ですがmmは分。DDは「年内の通算日」でddとは別物。「月＝大M、分＝小m」だけでも覚えておくと、定番のバグ「3月のはずが0時0分になる」を防げます。
05

文字列を日付に戻す

PARSING

フォーマットの逆、文字列を日付に変換するのがparseです。フォームに入力された「2024/03/05」のような文字列をLocalDateに戻す時に使います。format と parse は表と裏の関係です。

LocalDate.parse(text, formatter) 文字列 → 日付。書式が合わないと例外
// format と parse は逆向き
format
  • 日付 → 文字列
  • 表示するとき
  • date.format(f)
parse
  • 文字列 → 日付
  • 入力を受け取るとき
  • LocalDate.parse(s, f)
入力文字列と整形係の書式が食い違うと DateTimeParseException が出ます。
ParseExample.java
DateTimeFormatter f =
    DateTimeFormatter.ofPattern("yyyy/MM/dd");

// 文字列を日付に変換
LocalDate d =
    LocalDate.parse("2024/03/05", f);

System.out.println(d.getMonthValue()); // 3

// 書式が違うと例外になる
try {
    LocalDate.parse("2024-03-05", f); // "-"区切り
} catch (DateTimeParseException e) {
    System.out.println("書式が合いません");
}
// 入力の形と整形係をピタリ揃えるのが鉄則
POINT —
ユーザー入力を parse する時は必ず例外対策をセットで考えましょう。人は予想外の形で入力します。try-catch で受け止め、ダメなら「日付の形式が違います」とその人の言語で返す——これが国際化アプリの作法です。
06

ハマりどころ

COMMON PITFALLS

日付まわりの定番事故をまとめます。「記号ミス」「決め打ち書式」「古いDateとの混同」が御三家。ここを避ければ日付の国際化はほぼ安全です。

あるある事故と対策 新しい java.time を使うのが大前提
// 詰まりポイント
やりがちな失敗
  • mm と MM を取り違え
  • 全環境で同じ書式に固定
  • 古い Date / SimpleDateFormat
  • 戻り値を受け取らない
対策
  • 月=大M、分=小m
  • FormatStyle + Locale
  • java.time を使う
  • 計算結果を変数に受ける
新旧の使い分け
// ✕ 古い書き方（今は避ける）
// java.util.Date + SimpleDateFormat
// → スレッドで不具合・扱いづらい

// ○ 新しい書き方（Java 8以降の定番）
// java.time.* を使う
LocalDate d = LocalDate.now();
DateTimeFormatter f =
    DateTimeFormatter
      .ofLocalizedDate(FormatStyle.LONG)
      .withLocale(Locale.getDefault());
System.out.println(d.format(f));

// 結論：日付は java.time、表示は
// FormatStyle+Locale。これが現在の定番
