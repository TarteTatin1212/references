Properties編

PROPERTIES — KEY & VALUE FILES

01

プロパティファイルとは

WHAT IS .properties

プロパティファイルは「合言葉（キー）」と「中身（値）」をペアで書いておくテキストファイルです。拡張子は.properties。プログラムを書き換えずに、設定や文言だけを差し替えられるのが最大の利点です。

.properties 1行に1ペア、key=value だけのシンプルな世界
// messages.properties の中身イメージ
welcome=ようこそ
app.title=在庫管理システム
button.save=保存
max.count=100
左が「キー」、右が「値」。コードからは welcome という合言葉で「ようこそ」を取り出します。
なぜ外に出すと嬉しいのか
// もし文字をコードに直接書くと…
btn.setText("保存");
// 文言を変えるたびにコード修正＆再ビルド。面倒

// プロパティファイルに出しておけば…
btn.setText(props.getProperty("button.save"));
// 文言の変更はファイルを直すだけ
// → 翻訳者に渡せる
// → 英語版ファイルを別に用意すれば多言語化

// 「コードと文言を分ける」
// これが国際化の出発点です
POINT —
プロパティファイルは国際化専用ではありません。DB接続先・APIキー・各種設定など「コードに直書きしたくない値」全般の定番の置き場です。現場では超頻出。まず仲良くなっておきましょう。
02

書き方のルール

FILE FORMAT

ルールはとても少ないです。キー=値を1行ずつ書く。#で始まる行はコメント。これだけ覚えれば書けます。ただし細かい“クセ”がいくつかあるので、そこを押さえます。

書式の決まり = でも : でも区切れる／空白には注意
// 知っておきたいクセ
OK な書き方
  • key=value
  • key = value（空白OK）
  • key:value（: も可）
  • # コメント
ハマりやすい点
  • 値の末尾の空白も残る
  • 値は全部「文字列」扱い
  • キー名は大文字小文字を区別
  • 同じキーは後勝ち
config.properties
# これはコメント行
app.name=在庫管理
app.version=1.2

# = の代わりに : も使える
db.host:localhost

# 長い値は \ で次行に続けられる
notice=これは長い文章で \
次の行に続きます

# 値の「1.2」も「100」も文字列扱い
# 数値で使いたいなら自分で変換する
TIP —
値はすべて文字列（String）として読み込まれます。max.count=100 と書いても数値の100ではなく文字の"100"。数として使いたいときは Integer.parseInt(...) で変換する、と覚えておきましょう。
03

読み込む（load）

LOADING WITH Properties

プロパティファイルを読むにはPropertiesクラスを使います。手順は「①Propertiesを作る → ②loadで読み込む → ③getPropertyで取り出す」の3ステップ。型は決まり文句なので、まるごと覚えてOKです。

java.util.Properties load() で読み、getProperty() で取り出す
// 読み込みの流れ
① Properties の箱を用意する
② ファイルを開いて load() で流し込む
③ getProperty("キー") で値を取り出す
④ 無いキーには初期値も指定できる
出力結果：
アプリ名 = 在庫管理
バージョン = 不明（キーが無い場合）
LoadExample.java
import java.io.*;
import java.util.Properties;

// ① 箱を用意
Properties props = new Properties();

// ② ファイルを読み込む（try-with-resources）
try (InputStream in =
       new FileInputStream("config.properties")) {
    props.load(in);
}

// ③ キーで取り出す
String name = props.getProperty("app.name");
System.out.println("アプリ名 = " + name);

// ④ 無いキーの保険（第2引数＝初期値）
String ver = props.getProperty(
    "app.build", "不明");
System.out.println("バージョン = " + ver);
POINT —
getProperty("キー", "初期値") の第2引数は超便利。キーが存在しないとき null ではなく初期値を返してくれます。null チェック地獄を避けられるので、現場では積極的に使いましょう。
04

置き場所と読み込み元

WHERE TO PUT FILES

「ファイルが見つからない」は初学者の詰まりポイントNo.1。読み込み方法は大きく2つ、ファイルのパスで読む方法と、クラスパス（プログラムと一緒に運ばれる場所）から読む方法があります。違いを押さえましょう。

FileInputStream vs getResourceAsStream どこを基準にファイルを探すかが違う
// 2つの読み込み方
FileInputStream
  • 実行した場所が基準
  • 外部の設定ファイル向き
  • 後から差し替え可能
getResourceAsStream
  • クラスパスが基準
  • アプリ同梱の文言向き
  • jarの中でも読める
国際化の文言ファイルは「アプリと一緒に運ぶ」ので、後者のクラスパス読み込みが定番です。
2通りの読み込み
// 方法A：ファイルパスで読む
//   実行ディレクトリからの相対パス
try (InputStream in =
    new FileInputStream("config.properties")) {
    props.load(in);
}

// 方法B：クラスパスから読む（定番）
//   src/main/resources などに置いておく
try (InputStream in =
    MyApp.class.getResourceAsStream(
        "/config.properties")) {
    props.load(in);
}
// 先頭の「/」はクラスパスの先頭を意味する
// これを付け忘れて読めない、が定番事故
TIP —
getResourceAsStream("/config.properties") の先頭スラッシュを忘れると、クラスと同じパッケージ階層を探しに行ってしまい「見つからない」となりがち。クラスパスの一番上から探すなら先頭に /、と覚えておくと事故が減ります。
05

書き出す（store）

SAVING WITH store

読むだけでなく書き出しもできます。setPropertyで値をセットし、storeでファイルに保存。ユーザー設定の保存などに使います。読み込みの逆をやるだけなので難しくありません。

setProperty + store 値をセットしてファイルに書き出す
// 書き出した結果ファイル
user.properties
# ユーザー設定
#Wed May 27 10:00:00 JST 2026
theme=dark
lang=ja
fontSize=14
先頭2行はコメントと保存日時。Javaが自動で付けてくれます。
StoreExample.java
Properties props = new Properties();

// ① 値をセットする
props.setProperty("theme", "dark");
props.setProperty("lang",  "ja");
props.setProperty("fontSize", "14");

// ② ファイルに書き出す
try (OutputStream out =
    new FileOutputStream("user.properties")) {
    // 第2引数はファイル先頭のコメント
    props.store(out, "ユーザー設定");
}

// 「設定を保存する」ボタンの中身は
// だいたいこれ。覚えると応用が利く
POINT —
load と store はちょうど対になる操作です。「読む＝load／書く＝store」。さらに人間に読みやすいXML形式で保存する loadFromXML / storeToXML もありますが、まずは基本の2つで充分です。
06

ハマりどころ

COMMON PITFALLS

プロパティファイルでつまずく定番パターンを先に知っておきましょう。「読めない」「文字化けする」「値が変」の3つが御三家です。原因と対策をまとめます。

あるある事故と対策 先に知っておけば9割は防げる
// 詰まりポイント御三家
やりがちな失敗
  • ファイルが見つからない
  • 日本語が文字化け
  • 値の末尾に余計な空白
  • 数値のつもりが文字列
対策
  • 置き場所とパスを確認
  • UTF-8で読む / 保存する
  • = の後ろを詰める
  • parseInt で変換する
文字化け対策（UTF-8で読む）
// 昔のJavaは properties が英数字前提で
// 日本語が化けることがあった。
// Reader で文字コードを指定して読むと安全：

try (Reader r = new InputStreamReader(
        new FileInputStream("messages.properties"),
        StandardCharsets.UTF_8)) {
    props.load(r);   // Reader版のload
}

// 数値で使いたい値は変換する
String s = props.getProperty("max.count");
int max = Integer.parseInt(s.trim());
// trim() で前後の空白を落とすと安心
NEXT STEP —
プロパティファイルで「言葉を外に出す」感覚はつかめましたか。でも、これだけだと「日本語版」「英語版」を自分で選んで読み分ける必要があります。次のVol.3「ResourceBundle編」では、Localeを渡すだけで正しい言語のファイルを自動で選んでくれる魔法のような仕組みを学びます。ここからが国際化の本領発揮です。
