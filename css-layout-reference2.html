01

Flexboxの親子関係

FLEX CONTAINER & ITEMS

Flexboxには必ず「親（コンテナ）」と「子（アイテム）」の関係があります。 display: flexを指定するのは親だけ。すると直接の子要素が自動的にflexアイテムになります。 孫要素には影響しません。この「1段だけ」という感覚が超重要です。

flex-container + flex-items flexを書くのは親だけ / 子は自動でアイテムになる

/* HTML構造 */
/* <div class="container">  ← 親 */
/*   <div>A</div>  ← 子（アイテム）*/
/*   <div>B</div>  ← 子（アイテム）*/
/*   <div>C</div>  ← 子（アイテム）*/
/* </div>                          */

/* CSSは親にだけ書く */
.container {
  display: flex;  /* ← これだけ！ */

.container {
  display:         flex;

  /* 以下は書かなくても同じ */
  flex-direction:  row;       /* 横並び */

  align-items:     stretch;   /* 高さ揃え */

TIP — flexアイテムにはdisplayの種類（block/inline等）に関係なくwidth・heightが効きます。Vol.1で「spanにwidthが効かない」と学びましたが、display: flexの親を持つspanなら幅指定が通ります。これは覚えておくと超便利です。
03

flex-direction — 主軸の方向

FLEX DIRECTION

Flexboxには「主軸（main axis）」という概念があります。アイテムが並ぶ方向です。 flex-directionでこの方向を変えられます。横か縦か、順方向か逆方向か——4択です。

flex-direction row / column / row-reverse / column-reverse
// 4方向の並び比較（実際のflex）
row（デフォルト）

row-reverse
A
B
C
column
A
B
C
column-reverse
A
B
C
// reverseはHTMLの順番を変えずに見た目を逆にする 
.container {
  display: flex;

  /* 横並び（左→右）デフォルト */
  flex-direction: row;

  /* 横並び（右→左）*/
  flex-direction: row-reverse;

  /* 縦並び（上→下）*/
  flex-direction: column;

  /* 縦並び（下→上）*/
  flex-direction: column-reverse;
}

/* ━━ 主軸と交差軸 ━━━━━━━━━━━━━━━━
  direction: row の場合
    主軸   → 水平方向（横）
    交差軸 → 垂直方向（縦）

  direction: column の場合
    主軸   → 垂直方向（縦）
    交差軸 → 水平方向（横）

  justify-content は 主軸
  align-items     は 交差軸
━━━━━━━━━━━━━━━━━━━━━━━━━━━━ */
POINT — flex-direction: columnを使うと縦並びのflexになります。スマホ対応でよく使う定番パターンです。横並びのレイアウトをメディアクエリでcolumnに切り替えるだけでレスポンシブが完成します。またreverse系はHTMLの順番を変えずに表示順を逆にできるので、アクセシビリティ的にも有用です。

justify-content — 主軸の揃え方

JUSTIFY CONTENT

justify-contentは主軸方向のアイテム配置を制御します。 「左寄せ・右寄せ・中央・均等」など6つの値を覚えれば実務で困りません。 これだけで「ナビの間隔」「ボタンの位置」問題が全部解決します。

.container {
  display: flex;

  /* 左寄せ（デフォルト）*/
  justify-content: flex-start;

  /* 右寄せ */
  justify-content: flex-end;

  /* 中央寄せ */
  justify-content: center;

/* ★   space-evenly（メニュー等）   */
POINT — space-betweenはナビゲーションバーで最もよく使われる値です。「ロゴを左端、メニューを右端」はspace-between一発で完成します。space-aroundとspace-evenlyは似ていますが、端の隙間の計算方式が違います。端まできちんと揃えたいならspace-betweenが直感的です。

align-items — 交差軸の揃え方

ALIGN ITEMS

align-itemsは交差軸（主軸と垂直の方向）のアイテム配置です。 flex-direction: rowなら縦方向の揃え方になります。 「高さの違う要素を縦方向にどう揃えるか」——これで垂直中央寄せが実現できます。

.container {
  display: flex;

  /* 高さいっぱいに伸ばす（デフォルト）*/
  align-items: stretch;

  /* 上揃え */
  align-items: flex-start;

  /* 下揃え */
  align-items: flex-end;

  /* 縦方向に中央揃え ★★★ */
  align-items: center;

  /* テキストのベースラインで揃える */
  align-items: baseline;
}

/* ━━ justify と align の違い ━━━━━━━
  justify-content → 主軸（横）方向
  align-items     → 交差軸（縦）方向
  
  direction: row の場合
  justify = 横  /  align = 縦
  
  direction: column の場合
  justify = 縦  /  align = 横
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━ */
ハマりどころ ★★★ — align-items: stretch（デフォルト）は子要素の高さをコンテナいっぱいに引き伸ばします。「なぜかdivが異様に縦長になる！」と感じたらこれが原因です。align-items: flex-startを指定するとコンテンツの高さに戻ります。

gap — アイテム間の余白

GAP

gapはflexアイテム同士の間隔を一括指定できるプロパティです。 marginで調整するより断然スマートで、最初・最後のアイテムに不要な余白が付きません。 現代のCSSではgapが標準になっています。

row-gap:12px / column-gap:6px
.container {
  display: flex;

  /* 縦横同じ隙間 */
  gap: 16px;

  /* 縦と横を別々に */
  gap: 12px 24px; /* row col */

  /* それぞれ個別指定 */
  row-gap:    12px;
  column-gap: 24px;
}

/* NG: margin-right で調整（古い） */
.item { margin-right: 16px; }
.item:last-child {
  margin-right: 0; /* 最後だけ除外 */
}

/* ✅ gap ならこれだけ（シンプル）*/
.container { gap: 16px; }

/* gap は Grid でも使える（Vol.4）*/

align-self — 個別アイテムの縦揃え

ALIGN SELF

align-itemsがコンテナ全体の設定なのに対し、align-selfは特定の子要素1つだけの縦揃えを上書きできます。 「他は上揃えでも、このボタンだけ下に置きたい」という時に使います。

align-self 子要素に書く / 親のalign-itemsを個別上書き
// 親: align-items: flex-start / 子は個別指定
align-self
:flex-start
align-self
:center
align-self
:flex-end
align-self
:stretch
// 子要素側に書く → 親の設定を上書き 
/* 親: 全体の縦揃えを設定 */
.container {
  display:     flex;
  align-items: flex-start;
}

/* 子: 個別に上書き */
.item-a {
  align-self: flex-start; /* 上 */
}
.item-b {
  align-self: center;     /* 中央 */
}
.item-c {
  align-self: flex-end;   /* 下 */
}
.item-d {
  align-self: stretch;    /* 伸ばす */
}

/* 実用例：カードの「もっと見る」ボタン */
.card {
  display:        flex;
  flex-direction: column;
}
.card .btn {
  align-self: flex-end;
  margin-top: auto; /* 下に押し込む */
}
08

垂直中央寄せ 定番パターン

PERFECT CENTERING

CSSで長年の難問だった「縦横中央揃え」が、flexbox2行で解決できます。 これはCSSで最もよく使われる鉄板パターンです。必ず覚えてください。 ボタン内テキスト、カードの内容、ヒーロービジュアル——至る所で使えます。

perfect centering justify-content: center ＋ align-items: center = 完全中央
// 完全中央寄せ（実際のflex）
中央！
// 応用：ボタン内テキストを中央に
▶ 送信する
✕ キャンセル 
/* ★★★ 完全中央寄せの鉄板 ★★★ */
.center-box {
  display:         flex;
  justify-content: center; /* 横：中央 */
  align-items:     center; /* 縦：中央 */
}
/* ↑ この2行だけ！   */
/* ↑ 昔は苦労したのが嘘のよう */

/* ─ ボタンの中央揃え ─ */
.btn {
  display:         inline-flex;
  align-items:     center;
  gap:             6px;   /* アイコン+テキスト */
  padding:         10px 20px;
}

/* ─ 画面全体の中央 ─ */
body {
  display:         flex;
  justify-content: center;
  align-items:     center;
  min-height:      100vh;
}

/* ─ ナビバーの定番 ─ */
nav {
  display:         flex;
  justify-content: space-between;
  align-items:     center;
  padding:         0 24px;
  height:          60px;
}
ハマりどころ ★★ — 「justify-content: centerを書いたのに中央にならない！」——これはコンテナに幅（width）または高さ（height）が設定されていない場合によく起きます。コンテナがコンテンツと同じ幅・高さなら、中央寄せしても見た目が変わりません。width: 100%やmin-height: 100vhなど、親に広がりを持たせてから指定しましょう。
