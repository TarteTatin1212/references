01

flex-grow — 余白を分け合って伸びる

FLEX GROW

flex-growはコンテナに余白があるとき、その余白をどの比率で分け合って伸びるかを指定します。 デフォルトは0（伸びない）。1を指定した要素が余白を全て受け取り、 複数あれば数値の比率で分け合います。

flex-grow 余白の分配比率 / デフォルト0（伸びない）
// grow値による伸び方の違い（実際のflex）
全員 grow:0（デフォルト）→ 余白が余る

B

全員 grow:1 → 余白を等分して伸びる

grow:1
A:grow:2 / B:grow:1 / C:grow:1 → 2:1:1で分配

grow:2

grow:1
A:grow:1 のみ → Aだけが余白を全取り
A
grow:1

grow:0
C
grow:0
/* flex-grow の仕組み */
.container {
  display: flex;
}

/* デフォルト：伸びない */
.item { flex-grow: 0; }

/* 全員等分 */
.item { flex-grow: 1; }

/* 2:1:1 の比率で分配 */
.item-a { flex-grow: 2; }
.item-b { flex-grow: 1; }
.item-c { flex-grow: 1; }

/* ━━ よくある実用パターン ━━━━━━━━
   サイドバー固定 / メイン伸縮      */
.sidebar {
  flex-grow: 0;
  width: 240px;  /* 固定幅 */
}
.main {
  flex-grow: 1;  /* 残りを全部 */
}

/* ナビリンクが右端まで伸びる検索バー */
.search-input {
  flex-grow: 1;
  min-width: 0;  /* 後述の罠対策 */
}
02

flex-shrink — はみ出た分を縮む

FLEX SHRINK

flex-shrinkはflex-growの逆で、コンテナをはみ出した時に縮む比率を指定します。 デフォルトは1（縮む）。0にすると「絶対に縮まない＝固定幅」になります。 「アイコンが潰れる！」を防ぐのに必須の知識です。

flex-shrink はみ出た時の縮み比率 / デフォルト1（縮む）
// shrink:0 vs shrink:1（幅の狭いコンテナで比較）
❌ shrink:1（デフォルト）→ 全員が縮んで崩れる
ロゴ
shrink:1
長いテキスト
shrink:1
ボタン
shrink:1
✅ ロゴだけ shrink:0 → ロゴが潰れない
ロゴ
shrink:0
伸縮テキスト
shrink:1
CTA
shrink:0
// 「潰れてほしくない要素」に shrink:0 を付ける 
/* flex-shrink の仕組み */

/* デフォルト：縮む（はみ出たら圧縮） */
.item { flex-shrink: 1; }

/* 縮まない（固定サイズ確保）*/
.item { flex-shrink: 0; }

/* ━━ 実用パターン ━━━━━━━━━━━━━━━
   ナビバー：ロゴは固定、検索は伸縮  */
.nav-logo {
  flex-shrink: 0; /* 絶対縮まない */
  width: 120px;
}
.nav-search {
  flex-grow:   1; /* 余白を埋める */
  flex-shrink: 1; /* 縮んでもOK */
  min-width:   0; /* 後述の罠対策 */
}
.nav-btn {
  flex-shrink: 0; /* ボタンも固定 */
}

/* アイコン・アバター・バッジは
   shrink:0 が鉄則 */
.avatar {
  flex-shrink: 0;
  width:  40px;
  height: 40px;
}
03

flex-basis — 伸縮前の基準サイズ

FLEX BASIS

flex-basisはgrow/shrinkが働く前の基準サイズです。 widthと似ていますが、flexの文脈ではflex-basisが優先されます。 auto（デフォルト）はコンテンツ幅を基準にします。

flex-basis grow/shrink の起点となるサイズ / width より優先
// basis:auto vs basis:0 vs basis:固定値
basis:auto（デフォルト）→ コンテンツ幅が基準

basis:auto
少し長めテキスト

basis:0 + grow:1 → 余白を完全均等に分配
短い
basis:0
少し長めテキスト
basis:0

basis:0
basis:120px → 全員120pxから伸縮スタート
A
basis:120px
B
basis:120px
// basis:0はコンテンツ幅を無視して均等にできる 
/* flex-basis の値の種類 */

/* コンテンツ幅を基準（デフォルト）*/
.item { flex-basis: auto; }

/* 幅ゼロから伸縮（完全均等に最適）*/
.item { flex-basis: 0; }

/* 固定の基準幅を持たせる */
.item { flex-basis: 200px; }
.item { flex-basis: 30%; }

/* ━━ basis vs width ━━━━━━━━━━━━━━
  flex環境では flex-basis が width より
  優先される（basisがautoならwidthを参照）

  flex-basis: auto  → width に従う
  flex-basis: 0     → コンテンツ幅無視
  flex-basis: 200px → これが基準幅
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━ */

/* 完全均等カラムの鉄板 */
.col {
  flex: 1 1 0;
  /* grow:1 shrink:1 basis:0 */
}
04

flex ショートハンド早見表

FLEX SHORTHAND

flexプロパティはflex-grow・flex-shrink・flex-basisの3つをまとめたショートハンドです。 現場では個別指定よりショートハンドの定番値を覚えて使う方が実用的です。

flex: grow shrink basis 3つをまとめて指定 / 定番の書き方を覚える
// よく使う定番値と意味
書き方 展開後 使いどころ
flex: 1 1 1 0 均等伸縮。最もよく使う ★★★
flex: auto 1 1 auto コンテンツ幅を保ちつつ伸縮
flex: none 0 0 auto 伸縮しない完全固定 ★★
flex: 0 0 200px 0 0 200px 200px固定（サイドバー等）
flex: 2 2 1 0 他の2倍の幅を取る
flex:1
flex:none
flex:2
flex:1 / flex:none（固定）/ flex:2（倍幅） 
/* flex ショートハンドの定番 */

/* ★★★ 均等伸縮 */
.col { flex: 1; }
/* = flex-grow:1 shrink:1 basis:0 */

/* 伸縮しない固定 */
.icon { flex: none; }
/* = 0 0 auto */

/* 固定幅（サイドバー等）*/
.sidebar {
  flex: 0 0 240px;
  /* = grow:0 shrink:0 basis:240px */
}

/* メインは残りを全部 */
.main {
  flex: 1;
  min-width: 0; /* 後述の罠対策 */
}

/* 3カラム均等レイアウト */
.col-3 > * {
  flex: 1 1 0;
  /* basis:0でコンテンツ幅に左右されない */
}
TIP — 現場で最もよく見るのはflex: 1（均等伸縮）とflex: none（固定）の組み合わせです。 「残りの幅を全部くれ」→flex: 1、「絶対に縮まないでくれ」→flex: none、この2つを使い分けるだけで8割のケースに対応できます。
05

flex-wrap — 折り返し

FLEX WRAP

Flexboxはデフォルトで「絶対に1行に収めようとする」性質があります。 flex-wrap: wrapを指定すると、コンテナからはみ出た要素が次の行に折り返します。 レスポンシブなカード並びの基本テクニックです。

flex-wrap nowrap（デフォルト）/ wrap / wrap-reverse
// nowrap vs wrap の違い（幅200pxのコンテナ）
❌ nowrap（デフォルト）→ 全員を1行に押し込む
A
B
C
D
↑ はみ出ても無理やり1行
✅ wrap → はみ出たら次の行に折り返す
A
B
C
D
↑ 自然に折り返してくれる
/* flex-wrap の値 */
.container {
  display: flex;

  /* 1行に押し込む（デフォルト）*/
  flex-wrap: nowrap;

  /* 折り返しあり ★ */
  flex-wrap: wrap;

  /* 折り返し（逆順）*/
  flex-wrap: wrap-reverse;
}

/* ━━ レスポンシブカードの鉄板 ━━━━━
  flex-wrap + flex-basis で
  「幅が広ければ横並び、狭ければ縦積み」
  をメディアクエリなしで実現できる    */
.cards {
  display:   flex;
  flex-wrap: wrap;
  gap:       16px;
}
.card {
  flex: 1 1 280px;
  /* 280px以上なら横並び */
  /* 幅が足りなくなったら折り返す */
}

/* align-content で折り返し後の
   行間も調整できる */
.container {
  align-content: flex-start;
  /* center / space-between なども */
}
POINT — flex: 1 1 280px（basis:280px）＋flex-wrap: wrapは「コンテナが広い時は横並び、狭い時は縦積み」を自動で切り替えるレスポンシブの鉄板です。メディアクエリを書かなくてもカードグリッドが完成します。実際に次のセクションで使います。

実践パターン① ナビゲーションバー

NAVBAR PATTERN

Flexboxが最も輝く場面の一つがナビバーです。 「ロゴ左端・リンク中央・CTAボタン右端」という定番レイアウトがわずか数行で完成します。 grow/shrink/justify-content の組み合わせを実例で確認しましょう。

navbar space-between + align-items:center の定番構成

  • 機能
  • 料金
  • ドキュメント
無料で始める

  display:         flex;
  justify-content: space-between;
  align-items:     center;
  padding:         0 24px;
  height:          60px;
  background:      #111;
}

.nav-logo {
  flex-shrink: 0;
}

.nav-links {
  display:     flex;
  align-items: center;
  gap:         24px;
  list-style:  none;

.nav-cta {
  flex-shrink: 0;
}

  display: flex;
  gap:     16px;
}
.search {
  flex:      1;    /* 残りを全部 */
  min-width: 0;   /* 縮み対策 */
}

実践パターン② カードグリッド

CARD GRID PATTERN

flex-wrapとflex-basisを組み合わせると、レスポンシブなカード並びがメディアクエリなしで完成します。 さらにカード内でもflex-direction: columnを使うと「ボタンを常に下端に固定」できます。

card-grid flex-wrap:wrap + flex:1 1 280px でレスポンシブ対応
// カードグリッド（ボタン下端固定つき）
IMAGE
カードA
短めの説明文です。
詳しく見る →
IMAGE
カードB
こちらは少し長めの説明文が入っても大丈夫です。ボタンは常に下端に揃います。
詳しく見る →
IMAGE
カードC
説明文。
詳しく見る →
// ボタンが全カードで下端に揃っている 
/* ── カードグリッドのCSS ── */
.card-grid {
  display:   flex;
  flex-wrap: wrap;
  gap:       20px;
}

.card {
  flex:    1 1 280px; /* 280px基準で伸縮 */
  max-width: 360px;   /* 広がりすぎ防止 */

  /* ↓ カード内もflex（縦方向）*/
  display:        flex;
  flex-direction: column;
  gap:            8px;
}

/* 本文が伸びてボタンを下に押す */
.card-body {
  flex: 1; /* 余った空間を全部受け取る */
}

/* ボタンは常にカード下端 */
.card-btn {
  align-self: flex-end;
  margin-top: auto; /* さらに確実に下へ */
}

/* レスポンシブ動作：
  画面幅が広い → 横3列並び
  狭くなる    → 2列 → 1列
  ← メディアクエリ不要！ */

ハマりどころ：min-width: 0 問題

MIN-WIDTH TRAP

Flexアイテムにはデフォルトでmin-width: auto（コンテンツが収まる最小幅）が設定されています。 これが原因で「flex: 1を指定してもはみ出る」「テキストが切れない」という問題が起きます。 解決策はmin-width: 0を追加するだけです。

min-width: 0 flex:1 なのに縮まない → min-width:0 で解決 ★★★
// min-width の有無で変わる動作
❌ min-width指定なし → テキストがはみ出る
アイコン
とても長いテキストがここに入るとはみ出してしまう問題
✅ min-width:0 追加 → 正常に縮んで省略
アイコン
とても長いテキストがここに入るとはみ出してしまう問題
min-width: 0 → 縮める許可を与える
overflow: hidden → はみ出しを隠す
text-overflow: ellipsis → 「...」で省略 
/* ★★★ flex:1 なのに縮まない罠 ★★★ */

/* NG: こうなりがち */
.text-col {
  flex: 1;
  /* ← min-width: auto が残っている */
  /* ← コンテンツ幅より縮めない    */
}

/* ✅ OK: min-width:0 を追加 */
.text-col {
  flex:      1;
  min-width: 0; /* ← これだけ！*/
}

/* テキスト省略もセットで */
.text-ellipsis {
  flex:          1;
  min-width:     0;
  overflow:      hidden;
  white-space:   nowrap;
  text-overflow: ellipsis;
}

/* min-width:0 を使う典型ケース */
/* ・flex:1 なのにはみ出る      */
/* ・コードブロックが親を突き破る*/
/* ・長いURLが折り返さない      */
/* ・画像や表が縮まない         */
ハマりどころ ★★★ — 「flex: 1を書いたのに要素が縮まず親からはみ出す」——これはFlexboxで一番多い詰まりポイントです。min-width: autoというデフォルト値のせいで、コンテンツ幅より小さくなれない状態になっています。困ったらとりあえずmin-width: 0を試してください。これ一発で解決することが非常に多いです。
