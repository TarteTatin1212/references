01

グリッドラインを理解する

GRID LINES

Gridの配置を理解するには「ライン番号」の概念が必須です。 列や行の区切り線に番号が振られていて、その番号を使って要素の位置とサイズを指定します。 3列なら縦のラインは1〜4番（左端が1、右端が4）、負の番号は右端から数えます。

grid-lines 列ライン番号（3列 = ライン1〜4）/ 負の番号は右端から
// 3列グリッドのライン番号
1
2
3
4
col 1
col 2
col 3
row 2
row 2
row 2
-4 / 1
-3 / 2
-2 / 3
-1（右端）
// -1 は常に最後のライン（列数不問） 
/* 3列グリッドのライン番号 */
/*                         */
/*  1   2   3   4          */
/*  |   |   |   |          */
/*  [col1][col2][col3]      */
/*  -4  -3  -2  -1         */
/*  ↑ 負の番号は右から数える */

/* 要素を2列目から3列目に配置 */
.item {
  grid-column: 2 / 3;
  /* ライン2から始まりライン3で終わる */
  /* = 2列目のセル1つ分              */
}

/* 全幅に広げる（列数に関係なく） */
.full-width {
  grid-column: 1 / -1;
  /* 1（左端）から -1（右端）まで */
}

/* 行ラインも同じ考え方 */
/* 2行グリッドなら 行ライン 1〜3 */
.item {
  grid-row: 1 / 3;
  /* 行ライン1から3 = 2行にまたがる */
}
02

grid-column / grid-row — 位置の明示指定

GRID COLUMN & ROW

grid-column: 開始ライン / 終了ラインで要素の横方向の位置とサイズを指定します。 grid-rowは縦方向の制御です。この2つで「どのマス目に、何マス分置くか」が完全に決まります。

grid-column / grid-row 開始ライン / 終了ライン で位置を確定する
// 4列×3行グリッドでの明示配置
各要素に grid-column・grid-row を指定
A
col 1/3 row 1/2
B
col 3/5 row 1/3
C
col 1/2 row 2/4
D
E: col 2/4
F
// 要素が重なることもできる（z-indexで前後制御） 
.container {
  display: grid;
  grid-template-columns: repeat(4, 1fr);
  grid-template-rows: repeat(3, 80px);
  gap: 8px;
}

/* A: 1〜2列目、1行目 */
.item-a {
  grid-column: 1 / 3; /* 2列分 */
  grid-row:    1 / 2; /* 1行分 */
}

/* B: 3〜4列目、1〜2行目 */
.item-b {
  grid-column: 3 / 5; /* 2列分 */
  grid-row:    1 / 3; /* 2行分 */
}

/* 全幅（列数に関係なく右端まで）*/
.full {
  grid-column: 1 / -1;
}

/* 省略形：列だけ指定（行は自動）*/
.item {
  grid-column: 2; /* = 2 / 3 */
}

span — 「何マス分」で指定する

SPAN

ライン番号の代わりにspan Nを使うと「何マス分占めるか」で指定できます。 絶対的な位置より相対的なサイズで考えたいときに便利で、カラム数を変更しても崩れにくくなります。

grid-column: span N 「ライン番号」ではなく「何マス分」という指定
// span を使った配置パターン（4列）
span: 行き先ではなく「幅の広さ」で指定
span 2
2列分
auto
auto
auto
span 3（3列分）
auto
start ライン + span の組み合わせ
auto
2 / span 2
2列目から2列分
auto
1 / span 4（全幅）
/* span N: 何マス分かで指定 */

/* 2列分占める */
.item {
  grid-column: span 2;
  /* = 自動配置の位置から2列分 */
}

/* 2列目から始まって2列分 */
.item {
  grid-column: 2 / span 2;
  /* = ライン2から始まり2列占める */
  /* = grid-column: 2 / 4 と同じ  */
}

/* 行方向も同様 */
.item {
  grid-row: span 2; /* 2行分 */
}

/* ━━ span vs ライン番号 ━━━━━━━━━━━
  ライン番号 → 絶対配置。位置が変わると壊れる
  span      → 相対サイズ。列数変更に強い
  
  テンプレートを組む時 → span が柔軟
  厳密に位置を固定したい → ライン番号
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━ */

grid-template-areas — 名前付きエリア

GRID TEMPLATE AREAS

grid-template-areasは「部屋に名前をつけてレイアウトを図として書く」画期的な機能です。 CSS上でASCIIアートのように配置を描くので、コードを見ただけでレイアウトが一目でわかります。 チームでの開発や複雑なページ構造に特に威力を発揮します。

grid-template-areas レイアウトをASCII図で描く / grid-area で部屋を割り当て
// 名前付きエリアのプレビュー
header
sidebar
ナビ 01
ナビ 02
main
footer
"header header"
"sidebar main  "
"footer footer"
// コードがそのままレイアウト図になる 
/* ① 親：エリアの図を描く */
.page {
  display: grid;
  grid-template-columns: 240px 1fr;
  grid-template-rows: 60px 1fr 60px;
  grid-template-areas:
    "header  header"
    "sidebar main"
    "footer  footer";
  min-height: 100vh;
  gap: 0;
}

/* ② 子：エリア名を割り当てる */
header  { grid-area: header;  }
nav     { grid-area: sidebar; }
main    { grid-area: main;    }
footer  { grid-area: footer;  }

/* ━━ 覚えておくと便利 ━━━━━━━━━━━━
  . （ドット）→ 空白セル
  
  "header header"
  "sidebar ."    ← . = 空きセル
  "footer footer"
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━ */

/* レスポンシブ対応も直感的 */
@media (max-width: 768px) {
  .page {
    grid-template-columns: 1fr;
    grid-template-areas:
      "header"
      "main"
      "sidebar"
      "footer";
  }
}
POINT — grid-template-areasのもう一つの強みはレスポンシブ対応の書きやすさです。 メディアクエリ内でgrid-template-areasの図を描き直すだけで、サイドバーをメインの下に移動するなどの変更が直感的にできます。複雑なページほど威力を発揮します。

grid-auto-flow — 自動配置の方向

GRID AUTO FLOW

位置指定のない要素は自動配置アルゴリズムによって空きマスに流し込まれます。 grid-auto-flowでその方向（横→縦 or 縦→横）と、 隙間を詰めるdenseモードを制御できます。

grid-auto-flow row（デフォルト）/ column / dense

row（デフォルト）→ 横方向に流れ込む
span 2

4
5
6
row（隙間が生まれる）→ dense で解決
auto-flow: row
big
2
3
4
5
↑ 右に隙間
auto-flow: dense
big
2
3
4
5
↑ 隙間を詰める
.grid {
  display: grid;
  grid-template-columns: repeat(3, 1fr);

  /* 横方向に順番に配置（デフォルト）*/
  grid-auto-flow: row;

  /* 縦方向に順番に配置 */
  grid-auto-flow: column;

  /* 隙間を詰める（表示順は崩れる）*/
  grid-auto-flow: dense;

  /* column + dense の組み合わせ */
  grid-auto-flow: column dense;
}

/* dense の注意点 */
/* ・HTML の順番と表示順がずれる    */
/* ・スクリーンリーダーは HTML順    */
/* ・写真ギャラリーなど「順不同OK」*/
/*   な場合に使う                  */

/* 自動生成行の高さも指定できる */
.grid {
  grid-auto-rows: minmax(120px, auto);
  /* 最小120px・コンテンツで伸びる */
}

minmax() — 最小・最大幅を設定

MINMAX FUNCTION

minmax(最小値, 最大値)は「最低○○px確保しつつ最大○○frまで伸びる」という柔軟な幅指定ができます。 repeat(auto-fill, minmax(280px, 1fr))の組み合わせは、 レスポンシブカードグリッドの現代的な定番です。

minmax(min, max) 最小幅と最大幅を同時に指定 / auto-fill との合わせ技が強力
// minmax の動作（コンテナ幅変化に対応）
minmax(80px, 1fr) → 最低80px、余れば1frで伸びる
min:60
max:1fr
min:60
max:1fr
min:60
max:1fr
repeat(auto-fill, minmax(80px,1fr)) → 幅に応じて列数が変わる ★★★
A
B
C
D
E
← コンテナ幅に収まるだけ並べる
minmax(280px, 1fr) とは…
・幅280px以上あれば横に並ぶ
・280px未満になったら折り返す
・余った幅は1frで均等に伸びる 
/* minmax(最小, 最大) */

/* 最低100px・最大1fr */
.grid {
  grid-template-columns:
    minmax(100px, 1fr) minmax(100px, 1fr);
}

/* ★★★ レスポンシブカードの鉄板 */
.card-grid {
  display: grid;
  grid-template-columns:
    repeat(auto-fill, minmax(280px, 1fr));
  gap: 24px;
}
/* ↑ 280px確保できる分だけ列を作る
      余白は1frで均等に広げる
      メディアクエリ不要！ */

/* auto-rows にも使える */
.grid {
  grid-auto-rows: minmax(200px, auto);
  /* 行の最低高さを200pxに保ちつつ
     コンテンツが増えたら伸びる */
}

/* min-contentとmax-contentも使える */
.grid {
  grid-template-columns:
    minmax(min-content, 1fr);
}

auto-fit vs auto-fill の違い

AUTO-FIT VS AUTO-FILL

auto-fillとauto-fitはどちらも「収まるだけ列を作る」ですが、 アイテムが少ない時の余白の扱いが違います。 要素が少ない時に右端まで引き伸ばしたければauto-fit、 余白をそのまま残すならauto-fillです。

auto-fill vs auto-fit アイテムが少ない時だけ挙動が違う
// 3アイテムだけある時の比較
auto-fill → 空列が残る（右に余白）
A
B
C
auto-fit → 空列を折りたたみ・3アイテムが引き伸びる
A
B
C
アイテムが多い時 → 同じ動作
アイテムが少ない時 → 違いが出る

カード並び → auto-fill が自然
「常に幅いっぱい」→ auto-fit 
/* auto-fill: 空のトラックを保持 */
.grid {
  grid-template-columns:
    repeat(auto-fill, minmax(200px, 1fr));
}
/* アイテム3つ、4列分の幅がある場合：
   [A][B][C][空]  ← 空列が残る    */

/* auto-fit: 空トラックを折りたたむ */
.grid {
  grid-template-columns:
    repeat(auto-fit, minmax(200px, 1fr));
}
/* アイテム3つ、4列分の幅がある場合：
   [ A  ][ B  ][ C  ]  ← 3列に広がる */

/* ━━ どちらを使う？ ━━━━━━━━━━━━━
  カードグリッド（枚数増減する）
  → auto-fill が意図通りになりやすい
  
  「常にコンテナ幅いっぱいに広げたい」
  → auto-fit が適切
  
  迷ったら auto-fill から試す
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━ */

実践：雑誌風グリッドレイアウト

MAGAZINE LAYOUT

grid-columnとgrid-rowを使ったセルまたぎで、雑誌・ニュースサイト風の不均一なカード配置が作れます。 メインビジュアルを大きく、サブ記事を小さく——Gridでなければ作れないレイアウトです。

magazine-grid spanで大きさを変えた記事カードの配置
// 雑誌風 3列×3行グリッド（実際のGrid）
FEATURE
メイン記事
col 1-2 / row 1-2
サブ A
col 3 / row 1
サブ B
col 3 / row 2
記事 C
col 1 / row 3
記事 D
col 2 / row 3
記事 E
col 3 / row 3
// メインが 2列×2行分 を占有している 
.magazine {
  display: grid;
  grid-template-columns: repeat(3, 1fr);
  grid-template-rows: 300px 200px 200px;
  gap: 12px;
}

/* ヒーロー：2列 × 2行を占有 */
.hero {
  grid-column: 1 / 3; /* 1〜2列目 */
  grid-row:    1 / 3; /* 1〜2行目 */
}

/* サイド上：3列目・1行目 */
.side-a {
  grid-column: 3 / 4;
  grid-row:    1 / 2;
}

/* サイド下：3列目・2行目 */
.side-b {
  grid-column: 3 / 4;
  grid-row:    2 / 3;
}

/* 下段 3記事：自動配置に任せる */
.article-c,
.article-d,
.article-e {
  /* 位置指定なし → 3行目に自動配置 */
}

/* spanを使う書き方（同じ結果）*/
.hero {
  grid-column: 1 / span 2;
  grid-row:    1 / span 2;
}
ハマりどころ ★★ — 明示的に配置した要素と自動配置の要素が重なってしまうことがあります。 これは意図しないセルに自動配置されたときに起こります。 全要素にgrid-column/grid-rowを明示するか、grid-auto-flow: denseで詰め直すか、 あるいはgrid-template-areasで全エリアを宣言するかで回避できます。
