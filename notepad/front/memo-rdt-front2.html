FRONT.02 ── THYMELEAF CORE SYNTAX

対応する本編はVol.2（メモCRUD）。ここでは、その画面を題材にThymeleafの最頻出4構文を一つずつ手を動かして覚えます。th:text（表示）・th:each（繰り返し）・th:if（条件）・th:href（リンク）。この4つが分かれば、動的な画面の8割が作れます。式の書き方（${...}）も合わせて理解しましょう。

💡 つまずきやすいポイント先回り

① th:text はタグの中身を丸ごと置き換える（元のテキストは消える）／② th:each は繰り返したいタグ自体に書く（親ではなく）／③ 文字列は ${...} の中で + で連結／④ リンクは th:href="@{...}" と@{}を必ず使う。この4点を押さえれば大半は解決します。

01

式の書き方（${...}）

EXPRESSIONS

Thymeleaf の心臓部が式（expression）。${...} の中に、サーバーから渡されたデータへのアクセスを書きます。「メモのタイトル」なら ${memo.title}。まずこの記法に慣れましょう。

4種類の式記号で役割が違う
// 式の種類
記法意味
${...}変数（データ）
@{...}URL/パス
*{...}選択（フォーム・F3）
#{...}メッセージ（多言語）
// よく使うのは2つ

本シリーズでは ${...}（データ）と @{...}（パス）が主役。*{...} はフォーム（F3）で登場。

// データへのアクセス

${memo.title} は「memo オブジェクトの title」。ドットでプロパティをたどる。

// 式の書き方 // ─── ${...} 変数式（最頻出）─── ${memo.title} # memoのtitle ${memo.category.name} # ネストもOK ${memos} # リストそのもの ${message} # Controllerが渡した値 // ─── @{...} URL式 ─── @{/memos} # /memos へのパス @{/memos/{id}(id=${memo.id})} # /memos/5 のように組み立て // ─── 式の中で計算・連結 ─── ${memo.title + ' さん'} # 文字列連結 ${memo.pinned ? '📌' : ''} # 三項演算子 ${#lists.isEmpty(memos)} # ユーティリティ // ─── #temporals で日付整形 ─── ${#temporals.format( memo.updatedAt, 'yyyy-MM-dd HH:mm')} # → 2024-05-01 10:30 のように整形 ★ // ─── Controller との対応 ─── // Controller側: model.addAttribute("memo", memo); ↓ // テンプレート側: ${memo.title} # addAttributeの名前でアクセス ★★★
POINT — ${...} の名前は Controller の addAttribute と一致させる

${memo} の memo という名前は、Controller で model.addAttribute("memo", ...) と名前を合わせる必要があります。ここがズレると値が表示されない（フロント別冊F9で扱う頻出トラブル）。${memo.title} のドット記法は、内部的には getTitle() を呼んでいます（Lombokの@Dataが生成）。「渡した名前.プロパティ」が式の基本形です。

02

th:text で値を表示

TH:TEXT

最も基本の構文 th:text。タグの中身を、式の値で置き換える。<h1 th:text="${memo.title}"> なら、h1 の中身がメモのタイトルになります。HTMLエスケープも自動でしてくれます。

th:text の動き中身を差し替える
// 基本

th:text はタグの中身（テキスト）を置き換える。元々書いてあった文字は消えて、式の値になる。

// プレビュー
テンプレート: <h1 th:text="${memo.title}">仮タイトル</h1>
サーバー処理後: 会議メモ
// 自動エスケープ

th:text は < 等を自動でエスケープ。タグ文字を入れても安全に文字として表示（XSS対策）。

// th:text の使い方 // ─── 基本 ─── <h1 th:text="${memo.title}">仮タイトル</h1> # → 「仮タイトル」が memo.title に化ける ★ # → 「仮タイトル」は静的プレビュー用 // ─── 文字列を組み合わせる ─── <span th:text="${memo.title} + ' さん'"></span> # → 「会議メモ さん」 <span th:text="'全' + ${count} + '件'"></span> # → 「全3件」（文字列と数値を連結） // ─── 日付の整形 ─── <td th:text="${#temporals.format( memo.updatedAt, 'yyyy-MM-dd')}"></td> # → 「2024-05-01」 // ─── th:text と th:utext の違い ─── th:text → HTMLエスケープする（安全・標準）★ th:utext → エスケープしない（HTMLを埋める） # 例: 値が "<b>太字</b>" のとき th:text → <b>太字</b> と文字で表示（安全） th:utext → 太字 と太字で表示（XSS危険）★ # → 基本は th:text を使う ★★★
POINT — th:utext は原則使わない（XSS対策）

th:text は値を自動エスケープするので、ユーザーが入力した <script> 等もただの文字として安全に表示されます。これはXSS（クロスサイトスクリプティング）対策として重要。一方 th:utext はエスケープせずHTMLとして埋め込むので、ユーザー入力に使うとスクリプトが実行される危険がある。「基本は th:text、utext は信頼できる値だけ」。本シリーズでは th:text で統一します。

03

th:each で繰り返し

TH:EACH

一覧表示の主役 th:each。リストの要素ぶんだけ、タグを繰り返す。メモが3件あれば、<tr> が3行に増えます。「繰り返したいタグ自体」に書くのがコツです。

th:each の動き要素ぶん増える
// 基本の形

th:each="memo : ${memos}" ＝「memos の各要素を memo として取り出す」。memo は1件分の名前（自分で決める）。

// どこに書くか

繰り返したいタグ自体に書く。表の行なら <tr> に。親の <tbody> ではない。

// プレビュー
会議メモ
買い物リスト
Java学習

memos=[3件] なら 行が3つに

// th:each の使い方 // ─── 基本 ─── <tr th:each="memo : ${memos}"> <td th:text="${memo.title}"></td> </tr> # 読み:「memosの各要素を memo として」 # → memが3件なら <tr> が3行に ★★★ // ─── どのタグに書くか ─── NG: <tbody th:each="memo : ${memos}"> # 親に書くと… <tr>...</tr> </tbody> # → tbody ごと繰り返されてしまう ★ OK: <tbody> <tr th:each="memo : ${memos}"> # 行に書く ... </tr> </tbody> # → tr だけが増える（正しい）★★★ // ─── 番号やインデックスも取れる ─── <tr th:each="memo, stat : ${memos}"> <td th:text="${stat.count}"></td> # 1,2,3... <td th:text="${memo.title}"></td> </tr> # stat.count: 1始まりの番号 # stat.index: 0始まり / stat.first/last: 真偽 // ─── 空のときは th:if と組む（次節）─── # memosが空なら「メモがありません」表示
POINT — th:each は「増やしたいタグそのもの」に書く

初心者が必ず一度はやる間違いが、th:each を親タグに書いてしまうこと。表の行を増やしたいのに <tbody> に書くと、tbody ごと繰り返される。正しくは「繰り返したい単位のタグ（<tr>）に直接書く」。リスト項目なら <li>、カードなら <div class="card">。「1個分のタグに each を付けると、その個数ぶん増える」とイメージすると分かりやすいです。

04

th:if で条件表示

TH:IF

th:if は条件が真のときだけタグを表示します。「メモが空なら『ありません』」「カテゴリがあれば表示」など。逆の th:unless もあります。動的に出し分けるための道具です。

th:if の動き真なら表示
// 基本

th:if="${条件}" ＝条件が真ならタグを表示、偽ならタグごと消える。

// よく使う条件
  • ${memo.category != null}（存在チェック）
  • ${#lists.isEmpty(memos)}（空チェック）
  • ${memo.pinned}（真偽値そのまま）
// th:if と th:unless

th:if＝真なら表示、th:unless＝偽なら表示。「ある/ない」を出し分け。

// th:if の使い方 // ─── 空リストの案内 ─── <tr th:if="${#lists.isEmpty(memos)}"> <td colspan="3">メモがありません</td> </tr> # → memosが空のときだけ表示 ★ // ─── null チェック ─── <span th:if="${memo.category != null}" th:text="${memo.category.name}"></span> # → カテゴリがあるときだけ名前表示 # → null なのに .name すると落ちるので必須 ★★★ // ─── if と unless で出し分け ─── <span th:if="${memo.category != null}" th:text="${memo.category.name}"></span> <span th:unless="${memo.category != null}">未分類</span> # → ある→カテゴリ名 / ない→「未分類」★ // ─── 真偽値はそのまま条件に ─── <span th:if="${memo.pinned}">📌</span> # → pinned が true のとき📌表示 // ─── 三項演算子で短く（値の出し分け）─── <span th:text="${memo.pinned} ? '📌' : ''"></span> # → if/unless 2行を1行に ★
POINT — null の可能性があるものは th:if で守る

カテゴリ未設定のメモで ${memo.category.name} をいきなり書くと、category が null でエラーになります。だから th:if="${memo.category != null}" で存在を確かめてから表示する。これは本編で学んだ防御的プログラミングの画面版。「無いかもしれないものは、あるか確認してから触る」。null チェックを th:if で行うのは、画面づくりの基本動作です。

05

th:href でリンク

TH:HREF

リンクは th:href="@{...}" で組み立てます。固定パスもIDを埋め込む動的パスも作れる。@{/memos/{id}(id=${memo.id})} で /memos/5 のようなリンクに。@{}を必ず使うのが鉄則です。

th:href の組み立て@{}でパスを作る
// 固定リンク

th:href="@{/memos}" ＝ /memos へのリンク。シンプル。

// 動的リンク（IDを埋め込む）

@{/memos/{id}(id=${memo.id})} ＝ {id} の場所に memo.id を埋める。メモごとに違うリンク。

// クエリパラメータ

@{/memos(categoryId=${cat.id})} ＝ /memos?categoryId=3。括弧内がパラメータ。

// なぜ @{} か（F1復習）

公開パスが変わっても自動で正しく解決。直書きは禁物。

// th:href の組み立て // ─── 固定リンク ─── <a th:href="@{/memos}">一覧へ</a> # → /memos // ─── 動的リンク（パス変数）─── <a th:href="@{/memos/{id}(id=${memo.id})}" th:text="${memo.title}"></a> # memo.id=5 なら → /memos/5 ★★★ # {id} の場所に (id=...) の値が入る // ─── クエリパラメータ ─── <a th:href="@{/memos(categoryId=${cat.id})}"> このカテゴリで絞り込み </a> # cat.id=3 なら → /memos?categoryId=3 ★ // ─── 複数パラメータ ─── @{/memos(categoryId=${c.id}, page=${p})} # → /memos?categoryId=3&page=2 // ─── パス変数とパラメータの混在 ─── @{/memos/{id}/edit(id=${memo.id})} # → /memos/5/edit // ─── 鉄則（F1の復習）─── NG: href="/memos/5" # 直書き ★ OK: th:href="@{/memos/{id}(id=${memo.id})}" # → @{} で組み立てる ★★★
POINT — パス変数は @{/path/{x}(x=...)}、クエリは @{/path(x=...)}

@{} の中の書き分けがポイント：パスの一部（/memos/5）にするなら @{/memos/{id}(id=${memo.id})} とパスに {id} を置いて括弧で値を渡す。クエリ（?categoryId=3）にするなら @{/memos(categoryId=${cat.id})} とパスに置かず括弧だけ。「{}がパスに有る＝パス変数、無い＝クエリ」と覚えると整理できます。本編のURL設計（@GetMapping）と対応します。

06

一覧画面を組み立てる

BUILD THE LIST

学んだ4構文を全部使って、本編Vol.2 のメモ一覧画面を組み立てます。th:each で行を作り、th:text で値を出し、th:href でリンクを張り、th:if で空の案内。これが実際の list.html です。

memos/list.html の全体4構文の合わせ技
// 使う構文の確認
  • th:each：メモの数だけ行を作る
  • th:text：タイトル・日時を表示
  • th:href：詳細・編集へのリンク
  • th:if：メモが空のとき案内
// 完成イメージ
会議メモ ── 05-01 10:30 ── 編集/削除
買い物 ── 05-01 09:15 ── 編集/削除
// templates/memos/list.html（核心部） <table> <thead> <tr><th>タイトル</th> <th>更新日時</th> <th>操作</th></tr> </thead> <tbody> <!-- ① th:each で行を繰り返す --> <tr th:each="memo : ${memos}"> <!-- ② th:href リンク + th:text 表示 --> <td> <a th:href="@{/memos/{id}(id=${memo.id})}" th:text="${memo.title}">タイトル</a> </td> <!-- ③ 日付を整形して表示 --> <td th:text="${#temporals.format( memo.updatedAt, 'yyyy-MM-dd HH:mm')}"></td> <td> <a th:href="@{/memos/{id}/edit(id=${memo.id})}">編集</a> </td> </tr> <!-- ④ th:if で空のとき案内 --> <tr th:if="${#lists.isEmpty(memos)}"> <td colspan="3">メモがありません</td> </tr> </tbody> </table> # → 4構文の合わせ技で一覧が完成 ★★★ # → これが本編Vol.2 の list.html の核心
POINT — 4構文の組み合わせで「動的な一覧」の大半が作れる

この list.html には、th:each・th:text・th:href・th:if の4つが全部入っています。実は、動的な一覧画面の大半はこの組み合わせで作れます。「リストを繰り返し（each）、各項目の値を表示（text）、リンクを張り（href）、空なら案内（if）」。この型を覚えれば、メモ一覧もカテゴリ一覧も検索結果も同じパターンで組める。4構文は画面づくりの基礎体力です。

07

よくある書き間違い

COMMON MISTAKES

Thymeleaf 初心者が必ず一度はやる間違いを5つ。知っていれば数分でハマりを抜けられます。フロント別冊F9（困った時の画面編）の予告編でもあります。

あるある書き間違い 5選先に知れば回避できる

① th: を付け忘れる

  • text="${...}" だと普通の属性扱いで効かない
  • 対処：必ず th:text

② th:each を親タグに書く

  • tbody に書いて全体が繰り返される
  • 対処：繰り返す tr に書く（Section 03）

③ ${} の名前が addAttribute と不一致

  • 値が出ない・エラー
  • 対処：Controller の名前と合わせる

④ null を th:if なしで触る

  • category が null で例外
  • 対処：th:if で存在確認（Section 04）

⑤ リンクを @{} なしで直書き

  • 環境でパスがずれる
  • 対処：th:href="@{...}"（Section 05）
// あるある間違いの対処 // ─── ① th: 付け忘れ ─── NG: <h1 text="${memo.title}"> # → ただの謎の属性、表示されない ★ OK: <h1 th:text="${memo.title}"> // ─── ③ 名前の不一致 ─── # Controller: model.addAttribute("memoList", list); # テンプレート: NG: th:each="m : ${memos}" # 名前が違う ★ OK: th:each="m : ${memoList}" # 一致させる ★★★ // ─── ④ null を触る ─── NG: th:text="${memo.category.name}" # → category が null だと例外 ★ OK: th:if="${memo.category != null}" th:text="${memo.category.name}" // ─── ⑤ @{} なし ─── NG: <a href="/memos/5"> OK: <a th:href="@{/memos/{id}(id=${memo.id})}"> // ─── 良いThymeleafの条件 ─── ✓ th: を必ず付ける ✓ each は繰り返す単位のタグに ✓ ${} の名前は addAttribute と一致 ✓ null は th:if で守る ✓ リンクは @{} で組み立てる ★★★
POINT — 「表示されない」の9割は名前の不一致か th: 忘れ

「値が画面に出ない」というトラブルの大半は2つの原因：①th: を付け忘れた、②${名前} が Controller の addAttribute と一致していない。エラーも出ずに静かに表示されないので気づきにくい。困ったらまず「th: は付いているか」「名前は合っているか」を確認。これだけで多くが解決します。詳しいトラブルシューティングはフロント別冊F9 で扱います。

08

次巻への橋渡し

NEXT STEP

Thymeleaf の表示の4構文が身につきました。次は「ユーザーが入力する側」——フォームです。th:object・th:field・th:errors という、入力専用の構文に進みます。

F2 達成チェック表示の基本を習得
// 卒業チェックリスト
  • ${...} で値にアクセスできる
  • th:text で表示できる（utextは避ける）
  • th:each で繰り返せる（繰り返す単位に書く）
  • th:if で条件表示できる
  • th:href + @{} でリンクを組める
  • 4構文で一覧画面を組み立てられる
  • あるある間違いを知っている
// F2 で手に入れた4構文 ✓ "th:text — 値を表示（自動エスケープ）" ✓ "th:each — リストを繰り返す" ✓ "th:if — 条件で出し分け" ✓ "th:href @{} — リンクを組み立て" // この4つで動的画面の8割が作れる ★★★ // 次回 フロント別冊F3： // フォームと入力 // th:object / th:field / th:errors // 入力値の保持・エラー表示
NEXT STEP ── フロント別冊 VOL.F3 へ

F2 で「表示の4構文」を身につけ、データを画面に出せるようになりました。次はフロント別冊F3「フォームと入力編」。これまではサーバー→画面（表示）でしたが、F3では逆の画面→サーバー（入力）。th:object（フォーム全体）・th:field（各項目）・th:errors（検証エラー表示）を学びます。本編Vol.2 の作成フォーム、Vol.12 のバリデーションエラー表示が題材。入力値の保持という地味だけど重要なテクニックも登場します。フロント別冊F3 へ進みましょう。
