POSITION & STACKING — BLUEPRINT SERIES FINAL VOLUME

FlexとGridは「通常フローを賢く制御する」仕組みでした。 positionはそれとは異なり、要素を通常フローから切り離して自由な座標に置く機能です。 固定ヘッダー・バッジ・モーダル・追従サイドバー—— これらはpositionなしには作れません。 このVol.6でCSSレイアウトの全手法が揃います。

01

position の5種類を一望する

POSITION OVERVIEW

positionプロパティには5つの値があります。 「フローに残るか・フローから外れるか」という軸で整理すると覚えやすくなります。 static以外の全てでtop / right / bottom / leftの座標指定が有効になります。

position: static | relative | absolute | fixed | sticky フローに残る2種 / フローから外れる2種 / ハイブリッド1種
// フローとの関係で分類する
値 フロー 主な用途
static 残る デフォルト。top/left効かない
relative 残る 元の位置からずらす / absoluteの基点作り ★★★
absolute 外れる 最近傍のrelative祖先を基点に絶対配置 ★★★
fixed 外れる ビューポート基点で固定。スクロールで動かない ★★
sticky 両方 通常時はフロー内、スクロールで固定に切替 ★★
// static 以外は top/right/bottom/left が効く 
/* position の5値 */

/* デフォルト（何も変わらない）*/
.item { position: static; }

/* 元の位置からずらす */
.item { position: relative; }

/* 祖先のrelativeを基点に配置 */
.item { position: absolute; }

/* ビューポートを基点に固定 */
.item { position: fixed; }

/* スクロール中は流れ・端でstick */
.item { position: sticky; }

/* ━━ static 以外で使える座標 ━━━━━
  top: 20px;    / top: 50%;
  right: 0;     / right: 10%;
  bottom: 20px; / bottom: 0;
  left: 50%;    / left: auto;
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━ */
02

position: relative

RELATIVE

relativeは元の位置を基準にずらす指定です。 フローの中の「場所取り」はそのままで、見た目だけ動きます。 単体でずらす用途より、absoluteの基点を作るために使うことの方が圧倒的に多いです。

position: relative フローに残ったまま見た目だけずれる / absoluteの親に必須
// relative の動作（元の位置は空いたまま）
通常のブロック A
B: position:relative top:10px left:20px（ずれた）
通常のブロック C（Bが移動しても間は詰まらない）
// Bが動いてもCの位置は変わらない
relative の主な用途：
① absolute の基点を作る（★★★最重要）
② z-index の有効化
③ 微調整でほんの少しずらす 
/* relative の基本 */
.item {
  position: relative;
  top:  10px;  /* 元の位置から10px下へ */
  left: 20px;  /* 元の位置から20px右へ */
}

/* 元の「スペース」は残る
   → Cはずれない（Bの元の場所に空白）*/

/* ★★★ 最もよく使う用途：
   absoluteの基点（コンテナ）を作る */
.card {
  position: relative;
  /* ← これだけでOK（ずらさなくていい）
     中のabsolute要素がcard基点になる */
}

/* z-index を有効にする */
.above {
  position: relative;
  z-index: 1;
  /* position:staticにz-indexは効かない */
}
03

position: absolute

ABSOLUTE

absoluteはフローから完全に外れ、最も近いposition: static以外の祖先要素を基点に座標配置されます。 親にposition: relativeを付けることで「この要素を基点にして絶対配置する」という親子ペアが完成します。 バッジ・ツールチップ・装飾・ラベルなど用途が非常に広い技法です。

position: absolute relative親の四隅を基点に配置 / フローから外れる
// relative親 × absolute子 の組み合わせ
親: relative / 子: absolute → 四隅への配置
親要素（position: relative）
top:0
left:0
top:0
right:0
bottom:0
left:0
bottom:0
right:0
中央配置（top:50% + transform）
親（relative）
完全中央
// inset: 0 で親全体を覆うオーバーレイも作れる 
/* ★★★ relative + absolute の定番ペア */
.parent {
  position: relative; /* 基点を作る */
}
.child {
  position: absolute;
  top:    0;
  right:  0;
  /* 親の右上角に配置 */
}

/* 完全中央に置く */
.centered {
  position: absolute;
  top:       50%;
  left:      50%;
  transform: translate(-50%, -50%);
}

/* 親全体を覆うオーバーレイ */
.overlay {
  position: absolute;
  inset: 0;    /* top:0 right:0 bottom:0 left:0 */
  background: rgba(0,0,0,0.5);
}

/* 注意：absoluteは幅がコンテンツ分だけ */
.abs-item {
  position: absolute;
  width: 100%; /* 親幅いっぱいにしたい時 */
}
ハマりどころ ★★★ — 「absoluteが意図しない場所に飛んでしまう」——これはpositionで最も多いトラブルです。absoluteは最も近い「static以外の」祖先を基点にします。近くにrelativeがなければbodyやhtmlまで遡ります。必ず直接の親にposition: relativeを付けることを習慣にしてください。
04

position: fixed

FIXED

fixedはビューポート（ブラウザの表示領域）を基点に配置されます。 ページをスクロールしても動きません。固定ヘッダー・固定フッター・チャットボタン・クッキー通知—— 「スクロールしても常に見える要素」はこれで作ります。

position: fixed ビューポート基点 / スクロールしても動かない
// 固定ヘッダーのシミュレーション
⊞ BRAND（固定ヘッダー） スクロールしても動かない
コンテンツ行 01 — ページ本文
コンテンツ行 02 — ヘッダーの下に潜り込む
コンテンツ行 03 — padding-top で回避する
コンテンツ行 04 — z-index: 100 で前面に
注意：fixed はフローから外れる
→ body に padding-top:60px（ヘッダー高さ分）
  を付けてコンテンツが隠れないようにする 
/* 固定ヘッダーの定番 */
.header {
  position: fixed;
  top:    0;
  left:   0;
  right:  0;  /* 全幅 */
  height: 60px;
  z-index: 1000; /* 最前面 */
  background: #fff;
}

/* ヘッダー分の高さを補う */
body {
  padding-top: 60px;
}

/* 右下固定のCTAボタン */
.fab {
  position: fixed;
  bottom: 24px;
  right:  24px;
  z-index: 500;
}

/* クッキー通知バー（下部固定）*/
.cookie-bar {
  position: fixed;
  bottom:   0;
  left:     0;
  right:    0;
  z-index:  9999;
}
05

position: sticky

STICKY

stickyは通常のフロー内に存在しながら、スクロールで閾値に達すると固定されるハイブリッドです。 表のヘッダー行・セクション見出し・追従サイドナビなど、 「ページの一部だけ追従させたい」ときにfixedより自然に機能します。

position: sticky スクロールするまではフロー内 / 閾値でstickする
// ↓ スクロールしてみてください
コンテンツ上部
★ sticky ナビ（top:0 で吸い付く）
セクション内容 01
セクション内容 02
セクション内容 03
セクション内容 04
セクション内容 05
セクション内容 06
// 親がオーバーフローしてると動かないことがある
fixed との違い：
fixed → ページ全体を基準
sticky → 親要素の範囲内だけ追従 
/* sticky の基本 */
.sticky-header {
  position: sticky;
  top: 0;  /* ← 必須！閾値の指定 */
  z-index: 100;
  background: #fff;
}
/* top:0 がないと sticky が効かない */

/* テーブルの列ヘッダーを固定 */
table thead th {
  position: sticky;
  top: 0;
  background: white;
}

/* セクション見出しを追従 */
.section-title {
  position: sticky;
  top: 60px; /* 固定ヘッダー分下げる */
  z-index: 90;
}

/* ━━ sticky が効かない時の確認 ━━━━━
  ① top（or left等）が未指定
  ② 祖先に overflow:hidden/auto/scroll
  ③ 親の高さが子と同じ（余白がない）
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━ */

z-index — 重なり順の制御

Z-INDEX

z-indexは要素の前後の重なり順を指定します。数値が大きいほど前面に出ます。 ただしposition: staticの要素には効きません。またスタッキングコンテキストという落とし穴があります。

z-index 数値が大きいほど前面 / position:static には効かない
// z-index の重なり（実際のposition）
z-index:0
（後ろ）
z-index:1
z-index:2
（前面）
// よく使うz-index値の目安
通常コンテンツ　…… 0〜9
ドロップダウン　…… 100〜199
ヘッダー　　　　…… 200〜999
モーダル背景　　…… 1000〜1099
モーダル本体　　…… 1100〜1199
トースト通知　　…… 9000〜 
/* z-index の基本 */
.back  { position: relative; z-index: 0; }
.mid   { position: relative; z-index: 1; }
.front { position: relative; z-index: 2; }

/* position: static には z-index 効かない! */
.no-effect {
  /* position: static; ← デフォルト */
  z-index: 9999; /* ← 無視される */
}

/* 現場でよく使う値の目安 */
/* ヘッダー         */
.header  { z-index: 1000; }
/* モーダル背景     */
.overlay { z-index: 2000; }
/* モーダル本体     */
.modal   { z-index: 2001; }

/* ━━ スタッキングコンテキスト ━━━━━━
  opacity < 1 / transform / filter を
  付けると新しいスタッキングコンテキストが
  作られ、その中だけでz-indexが有効になる

  「z-index:9999 なのに前に出ない！」
  → 親のopacity/transformを確認
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━ */
ハマりどころ ★★★ — 「z-index:9999 にしたのに他の要素の後ろに隠れる！」——これはスタッキングコンテキストの罠です。親要素にopacity: 0.99・transform: translateZ(0)・filter: noneなどがあると、その中だけで新しいz-index空間が作られます。z-indexが効かない時は祖先のCSSプロパティを全て確認してください。
