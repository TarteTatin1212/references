01

CSSレイアウト全体地図

LAYOUT MAP

CSSには複数の「配置の仕組み」があります。どれを使うかを選ぶには、全体像を把握することが先決です。 このシリーズで学ぶ4つの柱を最初に頭に入れておきましょう。

layout-methods CSSレイアウト手法の全体像
// このシリーズで学ぶ配置の4本柱
▶ 通常フロー (Vol.1) 何も指定しない時の動作。ブロック・インライン・インラインブロック
▶ Flexbox (Vol.2–3) 1次元の並び。横一列・縦一列の配置を自在に制御
▶ Grid (Vol.4–5) 2次元の格子。行と列を同時にコントロール
▶ position (Vol.6) フローから切り離す。絶対配置・固定・スティッキー
// 全ての配置は「通常フローを土台にどう変えるか」という発想 
/* CSS配置の選び方ざっくり指針 */

/* ① 何もしない（通常フロー）*/
p, div, span {
  /* 特別な指定なし → 通常フローに従う */
}

/* ② 横並び・1方向の制御 */
.container {
  display: flex;    /* Flexbox */
}

/* ③ 行と列の2次元制御 */
.layout {
  display: grid;    /* Grid */
}

/* ④ フローから切り離す */
.fixed-header {
  position: fixed;  /* position */
}

/* ↑ どれを使うか迷ったら
   まずこの4択から選ぶ */
POINT — flexとgridはどちらも「通常フローを上書きする」仕組みです。だからこそ通常フローを知っているかどうかで、なぜ上書きする必要があるのかが分かるようになります。Vol.1はシリーズ全体の土台です。
02

通常フロー（Normal Flow）

NORMAL FLOW

CSSを何も指定しないとき、ブラウザは通常フローというルールで要素を並べます。 「ブロック要素は縦に積まれる」「インライン要素は横に流れる」——この2つが通常フローの全てです。

normal-flow 特別な指定なし → ブラウザが自動で並べる
// 通常フローの動作（実際のレンダリング）
<div> ブロック要素 — 行全体を占める
<p> ブロック要素 — 新しい行から始まる
↓ インライン要素は横に流れる
<span> テキスト <a> もっとテキスト <strong> が横に流れる
// ブロックは縦 / インラインは横 — これが大原則 
/* 通常フロー = 何もしない状態 */

/* ブロック要素（縦に積まれる） */
div, p, h1, ul,
header, section {
  /* display: block; が初期値 */
  /* → 幅は親いっぱい */
  /* → 前後に改行が入る */
}

/* インライン要素（横に流れる） */
span, a, strong, em {
  /* display: inline; が初期値 */
  /* → 文章の流れに沿って並ぶ */
  /* → width/height は効かない */
}

/* ↑ この挙動は
   CSSを書かなくても起きる
   = 「通常フロー」 */
03

ブロック要素の特性

BLOCK ELEMENT

ブロック要素は「1つで1行を丸ごと使う」要素です。横幅は親要素いっぱいに広がり、 次の要素は必ず下の行から始まります。div・p・h1〜h6・ul・li・sectionなどが該当します。

display: block 幅は親いっぱい / 前後に改行あり / width・height・margin全て効く
// ブロック要素のレンダリング
<div> width: 100%（自動）
<p> 前後に改行が入る
<div style="width:60%"> width指定が効く
// width: 60% も margin: auto も効く
margin: 0 auto → 中央寄せ 
/* ブロック要素の特性まとめ */

div {
  display: block;  /* デフォルト */

  /* ✅ 以下が全て効く */
  width:   60%;
  height:  200px;
  margin:  0 auto;  /* 中央寄せ */
  padding: 20px;
}

/* 主なブロック要素 */
/* div / p / h1〜h6    */
/* ul / ol / li       */
/* header / main      */
/* section / article  */
/* footer / nav       */
/* form / table       */

/* 幅をコンテンツに合わせたいなら */
div {
  width: fit-content;
  /* ← 幅がコンテンツ分だけに縮む */
}
TIP — margin: 0 autoでブロック要素を中央寄せできるのは、widthが指定されている場合のみです。widthが100%（デフォルト）のままだと中央寄せしても見た目が変わりません。「中央寄せが効かない！」と詰まったら幅の指定を確認してください。
04

インライン要素の特性

INLINE ELEMENT

インライン要素はテキストの流れに乗って横に並ぶ要素です。 widthとheightは効かず、上下のmarginも効きません。 span・a・strong・em・imgなどが代表例です。

display: inline 横に流れる / width・height無効 / 上下marginも無効
// インライン要素の制限
テキスト <span> width:100px → 無視される 続きのテキスト <a> また続く。行が折り返すと まるで文章の一部 のように振る舞う。
NG: width/height → 効かない
NG: margin-top/bottom → 効かない
OK: padding → 左右は効く（上下は見た目だけ）
OK: margin-left/right → 効く 
/* インライン要素の制限 */

span {
  display: inline;  /* デフォルト */

  /* ❌ 効かないもの */
  width:         100px;  /* 無視 */
  height:        50px;   /* 無視 */
  margin-top:    20px;   /* 無視 */
  margin-bottom: 20px;   /* 無視 */

  /* ✅ 効くもの */
  color:       red;
  background:  yellow;
  padding:     0 8px;   /* 左右OK */
  margin-left: 8px;    /* 左右OK */
  font-size:   14px;
}

/* 主なインライン要素 */
/* span / a / strong / em  */
/* code / label / button   */
/* img（特殊：サイズは効く）*/
ハマりどころ ★★★ — 「spanにwidthを指定したのに反映されない！」——これはCSSレイアウトで最も多いつまずきの一つです。インライン要素にwidthを効かせたいなら、display: inline-blockかdisplay: blockに変える必要があります。次のセクションで詳しく解説します。
05

インラインブロック

INLINE-BLOCK

display: inline-blockは「横に並ぶ」インラインの特性と 「width/heightが効く」ブロックの特性を合わせ持つハイブリッドです。 ナビゲーションのボタン並びや、横一列のカードによく使われます。

display: inline-block 横に並ぶ ＋ サイズが効く = ハイブリッド
// inline vs inline-block 比較
❌ display: inline（width指定しても縮まない）
A B C ← width:60px 無視
✅ display: inline-block（width・heightが効く）
A B C ← width/height有効
// ただしHTMLの空白文字が隙間になる罠あり（後述） 
/* inline-block の使い方 */

.nav-item {
  display:        inline-block;
  width:          120px;  /* 効く ✅ */
  height:         40px;   /* 効く ✅ */
  line-height:    40px;
  text-align:     center;
  vertical-align: top;
  /* ↑ ずれ防止の定番セット */
}

/* HTMLの空白で隙間ができる罠 */
/* <li>A</li>
   <li>B</li>  ← 改行=空白=隙間 */

/* 対処法① 親のfont-sizeを0に */
ul {
  font-size: 0;
}
ul li {
  display:   inline-block;
  font-size: 16px; /* 子で戻す */
}

/* 対処法② 今ならflexが楽 */
ul { display: flex; } /* Vol.2へ */
ハマりどころ ★★ — inline-blockの要素間に謎の隙間が生まれるのは有名な罠です。HTMLの改行・スペースが「空白文字」として扱われるためです。今では同じ目的でflexを使う方が隙間問題もなくシンプルです。ただしinline-blockは知識として必ず覚えておきましょう。
06

displayプロパティ早見表

DISPLAY PROPERTY

displayは要素の「見せ方の型」を切り替える最重要プロパティです。 全種類を覚える必要はありません。まずこの8つを押さえれば実務で困りません。

display: ... よく使う値 8種の早見表
// 使用頻度と用途
値 型 主な用途
block ブロック divをブロックに戻す・spanをブロック化
inline インライン divをテキスト扱いにする（稀）
inline-block インライン 横並び＋サイズ指定。ボタン・タグ等
flex フレックス ★★★ 横並び・縦並び。現代の定番
inline-flex フレックス flexだが外側はinline扱い
grid グリッド ★★★ 格子状レイアウト。ページ構造に
none — 要素を非表示（スペースも消える）
contents — 自身のボックスを消して子を直接出す 
/* display 切り替えの実例 */

/* aタグをボタン風にする */
a.btn {
  display:     inline-block;
  padding:     8px 20px;
  background:  #2b3d8f;
  color:       white;
}

/* liを横並びナビに */
nav ul {
  display: flex;  /* Vol.2 */
}

/* spanにwidthを効かせる */
span.tag {
  display: inline-block;
  width:   80px;
}

/* 要素を一時的に消す */
.hidden {
  display: none;
  /* ↑ visibility:hidden と違い
     スペースごと消える */
}

/* ページ全体のグリッド */
body {
  display: grid;  /* Vol.4 */
  grid-template-rows:
    auto 1fr auto;
}
07

マージンの相殺（ハマりどころ）

MARGIN COLLAPSE

マージンの相殺（Margin Collapse）は通常フロー特有の動作です。 縦方向に隣接する2つのブロック要素のmarginは、大きい方の値だけが適用されます。 「margin-bottomを20px指定したのに間が詰まってる！」と感じたらこれが原因です。

margin-collapse 縦marginは「合計」でなく「大きい方」になる
// 相殺の比較
❌ 想像（40px）
A（margin-bottom: 20px）
↑20px
↓20px
B（margin-top: 20px）
合計40px になると思いきや…
✅ 現実（20px）
A（margin-bottom: 20px）
20px（相殺）
B（margin-top: 20px）
大きい方（20px）だけ適用
/* マージン相殺の発生条件 */
/* ・通常フロー上の縦方向のみ発生  */
/* ・flexやgridでは相殺しない ✅   */

.box-a { margin-bottom: 20px; }
.box-b { margin-top:    20px; }
/* → 実際の隙間 = 20px（相殺） */
/* 40pxにはならない           */

/* ─ 回避方法 ─ */

/* 方法① 親にpadding/borderを付ける */
.parent {
  padding-top: 1px;
}

/* 方法② 親にoverflowを指定 */
.parent {
  overflow: hidden;
}

/* 方法③ そもそもflexにする */
.parent {
  display: flex;
  flex-direction: column;
  gap: 20px; /* 相殺なし */
}

/* 方法④ 片方のmarginだけ使う */
.box-a { margin-bottom: 40px; }
.box-b { margin-top: 0; }
ハマりどころ ★★★ — マージン相殺は親子間でも発生します。子要素のmargin-topが親要素に「漏れ出す」現象です。親にpadding-top: 1pxやoverflow: hiddenを付けると解消します。flexを使うとこの問題が根本から消えるのも、flex人気の理由の一つです。

配置手法の選び方チートシート

HOW TO CHOOSE

「flexとgrid、どちらを使えばいい？」「positionはいつ使う？」—— このシリーズ全体の判断軸を先に示します。迷ったらこの図を思い出してください。

layout-decision 配置手法の選択フロー
// 配置手法 決定木
要素を並べたい
↓ どんな並べ方？
横一列 or 縦一列
→ Flexbox (Vol.2–3)
行と列の格子
→ Grid (Vol.4–5)
↓ 特殊な配置？
スクロールしても固定
→ position: fixed
親の中で絶対座標
→ position: absolute
↓ 通常の文書の流れでOK
何も指定しない
→ 通常フロー（このVol）
/* 配置手法の使い分け早見表 */

/* ── 通常フロー（何もしない） ──
   文章・段落・見出しの流し込み  
   テキストと画像が混在する記事  
   シンプルな縦積みレイアウト    */

/* ── Flexbox ──────────────────
   ナビゲーションバー            
   ボタン群・タグ並び            
   カードの横一列                
   垂直中央揃え（超定番）        */

/* ── Grid ────────────────────
   ページ全体の骨格              
   カードの格子状並び            
   複雑な複数列レイアウト        
   header / sidebar / main       */

/* ── position ────────────────
   固定ヘッダー・フッター        
   モーダル・ポップアップ        
   バッジ・通知数字のオーバーレイ 
   スクロールで追従するサイドバー */

/* 迷ったら → まずFlexbox     */
/* 2次元必要 → Grid           */
/* フローから外れる → position */
